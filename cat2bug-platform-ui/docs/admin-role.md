# 角色管理

## 概述

角色管理是管理员用于创建、编辑和管理系统角色的功能模块。通过角色管理，管理员可以定义不同的权限集合，并将角色分配给用户，实现灵活的权限控制。

## 功能说明

### 角色列表

角色列表展示了系统中所有已定义的角色，包括：

- **角色名称**：角色的显示名称
- **角色标识**：角色的唯一标识符
- **角色描述**：角色的功能说明
- **权限数量**：角色拥有的权限总数
- **用户数量**：分配了该角色的用户总数
- **创建时间**：角色的创建日期
- **状态**：角色的启用/禁用状态

### 系统预置角色

系统默认提供以下角色：

- **系统管理员（admin）**：拥有系统的所有权限，可以管理团队、项目、用户和角色
- **团队管理员（team_admin）**：可以管理团队内的项目、成员和设置
- **项目管理员（project_admin）**：可以管理项目内的所有资源和成员
- **开发人员（developer）**：可以创建和修改缺陷、测试用例等
- **测试人员（tester）**：可以执行测试、提交缺陷
- **访客（guest）**：只读权限，可以查看项目信息

> **注意**：系统预置角色不能删除，但可以修改其权限配置。

### 创建自定义角色

管理员可以创建自定义角色：

1. 点击"新增角色"按钮
2. 填写角色信息：
   - 角色名称（必填）
   - 角色标识（必填，英文字母和下划线）
   - 角色描述
3. 配置角色权限：
   - 勾选需要授予的权限
   - 可以按模块展开查看详细权限
4. 点击"确定"完成创建

### 编辑角色

管理员可以编辑现有角色的信息和权限：

1. 在角色列表中找到目标角色
2. 点击"编辑"按钮
3. 修改角色信息或权限配置
4. 点击"确定"保存更改

> **提示**：修改角色权限后，拥有该角色的所有用户权限会立即生效。

### 删除角色

管理员可以删除自定义角色：

1. 在角色列表中找到目标角色
2. 点击"删除"按钮
3. 确认删除操作

> **注意**：
> - 系统预置角色不能删除
> - 删除角色前需要先将该角色从所有用户中移除
> - 删除操作不可恢复

### 角色分配

管理员可以为用户分配角色：

1. 进入"成员管理"模块
2. 选择目标用户
3. 点击"分配角色"
4. 选择一个或多个角色
5. 点击"确定"完成分配

> **说明**：一个用户可以拥有多个角色，用户的最终权限是所有角色权限的并集。

### 角色状态管理

管理员可以启用或禁用角色：

- **启用**：拥有该角色的用户可以正常使用角色权限
- **禁用**：拥有该角色的用户将暂时失去该角色的权限，但角色分配关系保留

## 权限说明

只有系统管理员（admin 角色）才能访问角色管理功能。

## 最佳实践

1. **最小权限原则**：只授予用户完成工作所需的最小权限
2. **角色命名规范**：使用清晰、有意义的角色名称
3. **定期审查**：定期审查角色权限配置，确保符合实际需求
4. **权限测试**：创建新角色后，建议先用测试账号验证权限是否正确

## 常见问题

**Q: 如何为新员工快速配置权限？**  
A: 创建符合岗位职责的角色模板，新员工入职时直接分配对应角色即可。

**Q: 用户拥有多个角色时，权限如何计算？**  
A: 用户的最终权限是所有角色权限的并集，即拥有所有角色的全部权限。

**Q: 修改角色权限后，需要用户重新登录吗？**  
A: 不需要，权限修改会立即生效。

**Q: 可以为不同团队的同一角色配置不同权限吗？**  
A: 角色是全局的，但可以通过创建不同的自定义角色来实现不同团队的权限差异化。

**Q: 如何查看某个用户拥有哪些权限？**  
A: 在成员管理中查看用户详情，可以看到用户的所有角色及对应权限。
