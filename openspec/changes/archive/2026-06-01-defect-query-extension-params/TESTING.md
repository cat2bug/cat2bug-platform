# 缺陷列表扩展查询参数 — 手工验收

## 前置

- 缺陷页已挂载「我的参与」统计块
- 项目内存在带日志的缺陷，热力图有 `count > 0` 的日期格

## 用例

### T1 参与 → 其它统计（清除扩展）

1. 点击热力图某日，确认列表仅为当日参与缺陷
2. 点击「我的待办」或其它统计块（状态/模块）
3. **预期**：网络请求 `params` 中无 `participationLogDate` / `participationUserId`

### T2 参与 → 重置（按 Tab）

1. 在「全部」Tab 点击热力图某日
2. 点击缺陷类型下拉**主按钮**（触发 `selectDefectTabHandle`，等价 `resetQueryByCurrentTab` 的 Tab 基准恢复）
3. **预期**：参与条件清除；列表恢复当前 Tab 默认范围（「全部」为未删除缺陷）

### T3 参与 → 切 Tab

1. 在「全部」Tab 点击热力图某日
2. 切换到某一自定义 Tab
3. **预期**：后续列表请求不含参与扩展键；结果符合该 Tab 配置

### T4 我的参与不叠加

1. 在列表上手动设置处理人/类型等（若 UI 支持）
2. 点击热力图某日
3. **预期**：切到「全部」Tab；先前手动条件不保留；仅参与日 + 当前用户筛选
