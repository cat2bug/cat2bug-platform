# 手工测试：登录页暗色主题改版（A + B + C）

## 前置

- 后端：`mvn spring-boot:run -pl cat2bug-platform-admin`
- 前端：`cd cat2bug-platform-ui && npm run dev`
- 参考稿：仓库根目录 `mockup_dark_v5.png`
- 浏览器：Chrome 或 Edge；准备暗色与浅色各测一轮

## A — 左栏与 token

1. 将应用主题设为 **dark**，打开 `http://localhost:2222/login`
   - 页面背景接近 mockup 纯黑（明显深于站内其他页 `#1e1e20`）
   - 左栏：Cat2 白、Bug 黄、-Platform 灰；Logo 下黄色短线
   - Hero + 三条 Feature 布局与 mockup 一致（图标黄色）
2. 语言切 **en_US** → Hero 为英文 slogan；切 **zh_CN** → 中文 slogan（非英文硬编码）
3. 将应用主题设为 **light**，再打开 `/login`
   - 左栏与表单仍可读，无未定义变量导致的白字白底
4. 确认刷新登录页后 **仍为** 用户所选 themeMode（未被强制改为 dark）
5. 视口宽度 &lt; 980px → 左栏隐藏，仅表单

## B — 像素鼠与右键

1. dark 主题下观察底部：老鼠为 **像素风灰鼠**（非写实棕鼠）
2. 不按键：老鼠自动横穿，多次观察方向 **左右随机**；会 **离开屏幕** 再从另一侧进入
3. 在底部条带 **按住右键** 移动 → 老鼠跟随；近底部更大、偏上更小
4. **松开右键** → 坐姿并左右看约 2–3 秒 → 恢复自动跑
5. 底部条带右键 → **无** 浏览器上下文菜单
6. 按住右键拖出条带后在页面空白处松键 → 不卡在跟鼠状态
7. macOS：Ctrl+左键 作为右键测试（若可用）
8. 系统「减少动态效果」开启 → 自动跑减弱或静止，登录功能正常

## C — 猫睡觉 z/Z

1. 猫插画上方仅 **小 z + 大 Z**（非三个字母轮播）
2. 观察 ≥5s：z 先出现，Z 延迟后上移，二者淡出并循环
3. 硬刷新：GLB 加载前（PNG fallback）与加载后均有 z/Z，位置在猫头附近
4. dark 下符号为浅色；light 下仍可见
5. 开启「减少动态效果」→ z/Z 静止、半透明

## 回归

1. 用户名密码登录成功 → 跳转首页
2. 验证码开启时登录流程正常
3. 语言旗帜切换后表单校验文案随语言更新
4. 控制台无 GLB / 精灵图加载报错

## 已知非本变更范围

- 右栏 Forgot password、Login 按钮箭头、去掉表单大标题：不验收
- 浅色主题像素级与 mockup 一致：不验收

## 精灵帧行说明

资源文件：`cat2bug-platform-ui/src/assets/images/mouse_sprite_pixel.png`（256×120，8 列×5 行，单帧 32×24px，透明底）。

| 行 | 常量 | 列数 | 用途 |
|----|------|------|------|
| row0 | `RUN` | 8 | 侧面跑动画；水平移动时取本行，`flipX` 表示向左 |
| row1 | `AWAY` | 4 | 远离（指针偏上 / dy 主导为负） |
| row2 | `TOWARD` | 4 | 靠近（指针偏下 / dy 主导为正） |
| row3 | `SIT` | 8 | 松右键后坐姿序列：停→坐直→看左→看右→回正等 |
| row4 | — | 可选 | 尾巴摆动等备用帧，当前逻辑未使用 |

调色：主体 `#9CA3AF`、轮廓 `#6B7280`、尾巴 `#D1A5A5`、高光 `#E5E7EB`。重生成：`python3 scripts/generate_mouse_sprite_pixel.py`（需 Pillow）。
