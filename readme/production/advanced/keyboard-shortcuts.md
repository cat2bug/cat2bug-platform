# 键盘快捷键 [键盘设置](/member/keyboard)

## 概述

Cat2Bug 提供一套**键盘优先**的操作方式，让您在熟悉键位后，尽量少用鼠标即可完成页面跳转和常用动作。快捷键采用「引导键 + 单字母」模型，避免与浏览器保留的 `Ctrl/Cmd + 字母` 组合冲突。

系统中有三种交互层次：

| 层次 | 操作方式 | 适用场景 |
|------|----------|----------|
| 引导键 | 按 `g` 或 **空格**，再按字母 | 跨页面导航、当前页主要动作 |
| 页面级 | **按住 ⌘（Mac）或 Ctrl（Windows/Linux）** | 工具栏按钮、表单字段、表格可见行 |
| 布局级 | **按住 ⇧⌘ 或 Shift+Ctrl** | 左侧菜单、顶栏工具、侧栏折叠、团队选择 |

> **说明**：下文统一写作 **⌘/Ctrl** 表示 Mac 的 Command 键或 Windows/Linux 的 Ctrl 键；**⇧⌘/Shift+Ctrl** 表示同时按住 Shift 与 ⌘/Ctrl。

默认键位可在 [键盘设置](/member/keyboard) 页查看和修改。若本文表格与设置页不一致，**以设置页当前显示为准**。

## 通用操作

### 引导键

| 按键 | 作用 |
|------|------|
| `g` | 打开**导航命令面板**，列出侧栏菜单与顶栏入口；2 秒内再按目标字母执行 |
| **空格** | 打开**当前页动作面板**（缺陷、通知、测试用例/计划、交付物、报告、文档及设置枢纽等页面支持） |
| `Esc` | 关闭命令面板；在下拉菜单中关闭菜单；在文档页可逐级退出导航或返回上一页 |

面板打开后，可在顶部搜索框输入关键字过滤条目，仍可用字母直达。

### 修饰键浮层

| 按键 | 作用 |
|------|------|
| 按住 **⌘/Ctrl** | 在当前页面控件右下角显示字母徽标，继续按字母即可触发 |
| 按住 **⇧⌘/Shift+Ctrl** | 在侧栏菜单、顶栏图标、侧栏折叠按钮上显示字母徽标 |
| 松开修饰键 | 徽标立即消失 |

通过 ⇧⌘/Shift+Ctrl 打开的下拉菜单（团队、语言、个人中心等）中，可用 **↑ / ↓** 切换选项，**Enter** 确认，**Esc** 关闭。

### 查询区键盘导航（← / →）

部分列表页查询条含**多个**筛选控件。当焦点在查询区内任一控件上时（含按 **S** 聚焦、或鼠标/Tab 点入），可用 **← / →** 在同一查询条内的控件间切换，**Esc** 退出查询导航态（焦点离开查询区也会自动退出）。

**查询区 ↔ 右侧工具栏**：焦点在最右（或唯一）查询控件时按 **→**，移至右侧工具栏第一个可见按钮；该按钮获焦后按 **←**，回到查询区最右控件。焦点在右侧工具栏内时，**← / →** 在工具栏各按钮间切换（首按钮 **←** 仍回到查询区）。单项查询页同样支持。

| 页面 | 查询控件（← → 顺序） |
|------|----------------------|
| 缺陷 | 类型 → 状态 → 处理人 → 名称/版本 →（→）工具栏首按钮 |
| 测试用例 | 用例编号 → 标题 → 级别 →（→）工具栏首按钮 |
| 测试计划 | 计划名称 → 版本 →（→）工具栏首按钮 |
| 项目报告 | 标题 → 时间范围 → 推送人 →（→）工具栏首按钮 |
| 文档管理 | 文档名 →（→）工具栏首按钮 |
| 交付物 | 模块名 →（→）工具栏首按钮 |
| 通知 | 标题 →（→）工具栏首按钮 |

下拉或日期面板已展开时，**← / →** 优先交给该浮层，不切换查询项。

### Split 下拉按钮（新建/导入/类型筛选等）

工具栏或查询条上的 **split-button**（左侧主文案 + 右侧箭头的组合按钮，如「+ 新建用例」）以及全站 **el-dropdown** 统一遵循：

| 操作 | 行为 |
|------|------|
| 焦点 | **整颗按钮**一个焦点环（不分别圈主按钮与箭头） |
| 快捷键 | **整颗按钮仅一个字母**（如用例页 **E** 打开新建菜单，**不再**为导入/导出单独分配 U/R） |
| **Enter** / **空格** / **↓**（焦点在按钮上） | 展开下拉菜单并聚焦首项 |
| **↑** / **↓**（菜单已展开） | 在菜单项间移动 |
| **Enter** / **空格**（菜单已展开） | 执行当前高亮菜单项，**并立即收起菜单** |
| **Esc** | 关闭菜单，焦点回到整颗按钮 |

> **说明**：菜单展开时系统仍会把 `.el-dropdown-menu` 视为浮层（用于屏蔽空格动作面板等），但 **不会** 因此禁用菜单内的 ↑↓ / Enter / Space / Esc；选中后菜单必须收起，焦点回到触发按钮。

适用页面示例：缺陷「新建」与「全部类型」、测试用例「新建用例」与「AI 创建」等所有 `split-button` 下拉。

### 附件上传（FileUpload）

表单中的**附件**上传（新建/编辑缺陷、用例、交付物、文档等）使用统一的 `FileUpload` 组件。Tab 至上传外框后：

| 操作 | 行为 |
|------|------|
| **Enter** / **Space** | 打开系统文件选择框（与点击「选取文件」相同） |
| **↓**（已有附件时） | 进入下方已上传文件列表，高亮第一项 |
| **↑** / **↓**（列表内） | 在已上传文件间切换高亮 |
| **Delete** / **Backspace**（列表内） | 删除当前高亮文件 |
| **↑**（列表首项） | 回到上传外框（选取文件 / 拖拽区） |
| **Tab** | 一次跳至下一个表单字段 |

用例 Excel **导入**弹框的上传区遵循相同 ↑↓ / Delete 规则（另见 [导入用例](../user-guide/current-project/case/case-import.md#键盘操作)）。

### 工具栏按钮焦点（显示字段等）

列表页工具栏上的 **plain 图标按钮**（如「显示字段」列选择、批量操作等）聚焦时 MUST **仅显示一层**焦点环：

- 使用 `box-shadow` 单环（`cat2bug-unified-focus-ring`），**禁止** `outline` + `outline-offset` 与 Element 按钮 `:focus` 改色 border 叠加成「双边框」
- 键盘 **← / →** 导航（`list-query-toolbar-nav-focused`）与鼠标/Tab 聚焦使用**同一套**单环样式
- 查询区内输入框聚焦仍由内部 `el-input` 的 `:focus` 绘制单环，外层 `list-query-nav-item` 不再额外描边

### 何时快捷键不会触发

以下情况系统**不会**拦截按键，以便正常输入或操作：

- 焦点在输入框、文本域、下拉框或 Excel 单元格编辑态
- 正在使用输入法组合输入（IME）
- 焦点在按钮或链接上时，**空格**不会打开动作面板
- 已打开对话框、抽屉、消息框或下拉浮层（如成员选择、日期面板）时，**空格**不会打开页面动作面板
- 按住 ⌘/Ctrl 的页面徽标在弹框/抽屉打开时也会隐藏

登录页的老鼠 D-pad 操控（⌘/Ctrl + 方向键）为登录页专属装饰交互，不走全局引导键引擎。

## 不可绑定键 / 浏览器冲突

Cat2Bug 采用「引导键 + 单字母」与「按住 ⌘/Ctrl + 单字母」模型，刻意**不**映射 `⌘/Ctrl + 字母` 组合键，以免与浏览器、操作系统保留快捷键冲突。部分单字母在特定场景下**禁止绑定**，键盘设置页保存时会拒绝并提示原因。

### 为何部分字母不可自定义

Web 页面无法可靠拦截下列组合；若强行映射会破坏用户预期（关闭标签、刷新、新建窗口等）。实现上由 `reserved-keys.js` 统一维护禁用池，设置页、页面动作浮层、表单字段浮层与表格行动态分配共用同一数据源。

### 单字母禁用池（按场景）

| 场景 | 禁用字母 | 典型原因 |
|------|----------|----------|
| 页面动作 / 表格行动态 | **A、C、M、N、Q、T、V、X、Z** | 浏览器编辑键（A/C/V/X/Z）+ 新标签/新窗口/退出（N/T/Q）+ 最小化（M）等 |
| 表单字段浮层 | 上列 + **W、R、L、H** | 关闭标签（W）、刷新（R）、地址栏（L）、隐藏应用（H） |
| 键盘设置页 | 上述并集 | 用户不可将保留字母保存为自定义键位 |

> **说明**：部分动作（如测试用例「AI 创建」默认 **I**、交付物「展开/折叠」默认 **T**）仍可通过 **空格** 动作面板触发；保留字母（如 A/C/V）通常**不会**出现在按住 ⌘/Ctrl 的页面徽标上。以界面实际显示的徽标为准。

### 常见浏览器 / OS 保留组合

以下组合由浏览器或操作系统处理，应用**不会**拦截：

| 组合（Mac ⌘ / Win Ctrl） | 系统行为 |
|--------------------------|----------|
| **+W** | 关闭当前标签页 |
| **+Shift+W** | 关闭窗口（Safari 等） |
| **+T** | 新建标签页 |
| **+N** | 新建窗口 |
| **+Shift+N** | 无痕窗口（Chrome） |
| **+Q** | 退出应用（macOS） |
| **+R / F5** | 刷新页面 |
| **+Shift+R** | 硬刷新 |
| **+L / Alt+D** | 聚焦地址栏 |
| **+F** | 页内查找 |
| **+P** | 打印 |
| **+S** | 保存网页 |
| **+A / C / V / X / Z** | 全选 / 复制 / 粘贴 / 剪切 / 撤销 |
| **+1 … 9** | 切换标签（Chrome/Edge） |
| **+[ / ]** | 后退 / 前进（Chrome、Safari 等） |

**日期范围面板**（报告/缺陷等查询条）打开时：**`,` / `.`** 切换月份，**`-` / `=`** 切换年份（**`[` / `]`** 已禁用，避免与浏览器后退/前进冲突）。

**导航与文档的取舍示例：**

- 侧栏「文档管理」导航用 **`g` → F**，而非 W（避免与 **⌘/Ctrl+W** 关闭标签冲突）。
- 报告「新建」默认 **E**（**N** 为保留键，**⌘/Ctrl+N** 交给浏览器新建窗口）。
- 登录提交等不得文档化为「⌘/Ctrl+L」；Safari 会聚焦地址栏，以屏上徽标为准。

## 自定义快捷键

### 入口

1. 点击右上角头像
2. 选择 **键盘设置**，或按 `g` → `U` → `K`
3. 也可直接访问 [/member/keyboard](/member/keyboard)

### 可配置项

- **总开关**：关闭后全局快捷键不再拦截按键
- **引导键**：导航（默认 `g`）、动作（默认空格）
- **分组键位**：导航、各页面动作、布局级等，点击字母可重新绑定
- **冲突检测**：同组内重复字母会标红提示；保留字母（见 [不可绑定键 / 浏览器冲突](#不可绑定键--浏览器冲突)）无法保存
- **恢复默认**：支持单项或全部恢复

### 存储说明

设置保存在**当前浏览器**的本地存储（localStorage）中，**不会**同步到其他设备或浏览器。换电脑或清空站点数据后需重新配置。

## 常见问题

**Q: 按空格没有打开动作面板？**  
A: 请确认：(1) 当前页面已支持动作面板（如缺陷列表）；(2) 未打开抽屉、对话框或下拉菜单；(3) 焦点不在输入框或按钮上；(4) 键盘设置中总开关已开启。

**Q: 按住 ⌘/Ctrl 看不到字母徽标？**  
A: 弹框或抽屉打开时页面徽标会隐藏；请先关闭浮层。Mac 请按 Command，Windows 请按 Ctrl。若仍无显示，检查键盘设置是否关闭总开关。

**Q: 在 Excel 模式编辑单元格时快捷键无效？**  
A: 这是预期行为。编辑单元格时按键留给表格输入；结束编辑后再使用快捷键。

**Q: 与浏览器快捷键冲突怎么办？**  
A: 导航与页面动作使用「引导键 + 单字母」，刻意避开 `⌘/Ctrl+W/T/N` 等浏览器组合；详见 [不可绑定键 / 浏览器冲突](#不可绑定键--浏览器冲突)。布局级使用 ⇧⌘/Shift+Ctrl，仍可能与少数浏览器扩展冲突，可在键盘设置中改字母（保留字母除外）。

**Q: 表格行的字母为什么和文档写的不一样？**  
A: 行号徽标按**当前可见行**动态分配，不是固定映射。以按住 ⌘/Ctrl 时界面上显示的字母为准。

**Q: 修改键位后文档还有效吗？**  
A: 本文描述**默认**键位。自定义后以 [键盘设置](/member/keyboard) 页显示为准。
