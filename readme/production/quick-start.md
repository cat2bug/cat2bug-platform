# 快速开始

本指南将帮助您快速上手 Cat2Bug 系统，完成从登录到创建缺陷的完整流程。

## 整体流程

使用 Cat2Bug 进行缺陷管理的基本流程如下：

```
┌─────────────┐
│ 首次安装配置 │ (仅首次运行需要)
└──────┬──────┘
       ▼
┌─────────────┐
│  登录系统    │
└──────┬──────┘
       ▼
┌─────────────┐
│  创建团队    │
└──────┬──────┘
       ▼
┌─────────────┐
│ 添加团队成员 │
└──────┬──────┘
       ▼
┌─────────────┐
│  创建项目    │
└──────┬──────┘
       ▼
┌─────────────┐
│添加项目成员  │
└──────┬──────┘
       ▼
┌─────────────┐
│创建测试用例  │
│  (可选)     │
└──────┬──────┘
       ▼
┌─────────────┐
│ 创建测试计划 │
│  (可选)     │
└──────┬──────┘
       ▼
┌─────────────┐
│  创建缺陷    │
└──────┬──────┘
       ▼
┌─────────────┐
│  团队协作    │
└─────────────┘
```

**流程说明：**

0. **首次安装配置** - 首次部署系统时，访问系统将重定向到「首次安装向导」，根据提示完成数据库、缓存及超级管理员账户的配置，完成后重启后台服务进程即可。
1. **登录系统** - 使用超级管理员或注册分配的账号密码登录系统
2. **创建团队** - 创建一个团队作为协作单位
3. **添加团队成员** - 邀请或创建团队成员（必须步骤，项目成员只能从团队成员中选择）
4. **创建项目** - 在团队下创建项目
5. **添加项目成员** - 从团队成员中选择项目参与者
6. **创建测试用例**（可选）- 编写测试用例，规范测试流程
7. **创建测试计划**（可选）- 制定测试计划，组织测试活动
8. **创建缺陷** - 在项目中记录和管理缺陷
9. **团队协作** - 分配、处理和跟踪缺陷

下面将详细介绍每个步骤的操作方法。

## 首次安装配置（仅首次启动需要）

如果是首次部署并运行系统，访问地址 `http://localhost:2222` 将会自动重定向至 **首次安装向导 (`/setup`)**。

1. **数据库配置**：支持 H2（默认，免安装）或 MySQL，填写好连接信息后可点击「测试连接」。
2. **缓存配置**：支持 Caffeine 内存缓存（默认）或 Redis，填写完毕后可点击「测试连接」。
3. **存储与日志配置**：配置文件上传保存的目录和系统运行日志保存的目录。
4. **AI 配置（可选）**：配置 Ollama 本地大模型服务地址。
5. **超级管理员配置**：自定义超级管理员的账号与密码（默认为 `admin` / `cat2bug`），建议在生产环境重新设置一个安全的密码。
6. **安装并重启**：在最后一页确认配置无误后，点击「完成安装」。**请注意，点击提交后必须在控制台或服务器上重启后台 Java 进程**，新数据库、缓存和安全配置才能完全加载。

*注：在 Docker 部署中，如果已配置 `CAT2BUG_INSTALL_SKIP=true`，系统将直接跳过向导。*

## 登录系统

1. 访问系统地址：`http://localhost:2222`（已完成安装向导并重启后）
2. 输入用户名和密码（或新注册的普通用户账号）
3. 点击登录按钮

## 创建团队

在创建项目之前，需要先创建一个团队：

1. 点击左侧菜单"团队管理"
2. 点击"新增"按钮
3. 填写团队信息：
   - 团队名称
   - 团队介绍（可选）
4. 点击"确定"保存

## 添加团队成员

创建团队后，必须添加团队成员，因为项目成员只能从团队成员中选择：

1. 进入团队详情页
2. 点击"成员管理"
3. 添加成员的方式：
   - **邀请现有用户**：输入用户账号或邮箱，发送邀请
   - **创建新成员**：填写成员信息，系统自动创建账号
4. 设置成员角色和权限
5. 点击"确定"保存

> **重要提示**：项目成员必须先是团队成员，所以在创建项目前务必先添加团队成员。

## 创建项目

添加团队成员后，就可以在团队下创建项目：

1. 点击左侧菜单"项目管理"
2. 点击"新增"按钮
3. 填写项目信息：
   - 选择所属团队
   - 项目名称
   - 项目介绍（可选）
   - 项目图标（可选）
4. 点击"确定"保存

## 添加项目成员

创建项目后，需要为项目添加成员：

1. 进入项目详情页
2. 点击"项目设置" → "成员管理"
3. 从团队成员列表中选择项目参与者
4. 为每个成员分配项目角色：
   - 项目管理员
   - 开发人员
   - 测试人员
   - 观察者
5. 点击"确定"保存

> **注意**：只能从团队成员中选择项目成员，如果需要添加新成员，请先在团队中添加。

## 创建测试用例（可选）

为了规范测试流程，可以创建测试用例：

1. 进入项目详情页
2. 点击左侧菜单"测试用例"
3. 点击"新增"按钮
4. 填写用例信息：
   - 用例标题
   - 前置条件
   - 测试步骤
   - 预期结果
   - 优先级
5. 点击"保存"

## 创建测试计划（可选）

可以创建测试计划来组织测试活动：

1. 进入项目详情页
2. 点击左侧菜单"测试计划"
3. 点击"新增"按钮
4. 填写计划信息：
   - 计划名称
   - 测试范围
   - 开始/结束时间
   - 关联测试用例
   - 指派测试人员
5. 点击"保存"

## 创建缺陷

1. 进入项目详情页
2. 点击"新增缺陷"按钮
3. 填写缺陷信息
4. 上传截图（可选）
5. 点击"提交"

## 团队协作

1. 在项目中添加团队成员
2. 分配缺陷给相应成员
3. 成员可以更新缺陷状态
4. 通过评论进行沟通
