# 测试用例 [/project/case](/project/case)

测试用例是软件工程及 BUG 管理中不可或缺的元素，在实际工作中，编写测试用例又一直消耗测试人员大量的时间和精力。

因此，Cat2Bug-Platform 从 0.1.0 开始，将支持测试用例的使用，并在后续工作中将提高测试用例的质量、减少测试用例的维护时间为目标，不断优化系统及使用方式，为大家提供更为轻松便捷的测试环境。

## 名词解释

| 名词 | 说明 |
|------|------|
| 测试用例 | 为验证软件是否满足特定需求而设计的一组‌测试输入、执行条件和预期结果‌的文档化描述 |
| 交付物 | 软件（系统功能模块、组件等）、硬件产出物 |
| 预期 | 执行测试后期望得到的结果 |
| 等级 | 测试用例的优先级，分为 P0（核心）、P1（重要）、P2（一般）、P3（次要）、P4（最低） |
| 前置条件 | 执行测试前需要满足的条件或准备工作 |
| 步骤 | 执行测试的详细操作步骤，包括步骤描述、步骤预期 |

## 键盘快捷键

通用说明见 [键盘快捷键](../../advanced/keyboard-shortcuts.md)。

在测试用例列表页（无弹框/抽屉/下拉时）可用 **空格** 打开动作面板，或按住 **⌘/Ctrl** 查看工具栏徽标：

| 字母 | 动作 |
|------|------|
| S | 聚焦查询（**← / →** 在编号、标题、级别间切换；最右项 **→** 进入右侧工具栏首按钮，**Esc** 退出） |
| E | 新建用例菜单（展开后 **↑↓** 选新建/导入/导出，**Enter** 或 **空格** 执行并收起菜单，**Esc** 关闭） |
| I | AI 创建菜单（同上；**I** 避免与浏览器 ⌘/Ctrl+A 全选冲突） |
| D | 批量删除（需已勾选） |
| M | 切换模块树（主要通过空格面板） |
| G | 模块树导航（树可见时；↑↓ 移动，Enter 选中，Esc 退出） |
| B | 上一页 |
| P | 下一页 |

按住 **⌘/Ctrl** 时，**当前可见行**用例编号列会显示动态字母（优先 `1`–`9`），按字母打开该用例编辑。

**查询区**：焦点在查询条内时，**← / →** 依次切换「用例编号 → 标题 → 级别」；最右一项 **→** 进入右侧工具栏，工具栏内 **← / →** 依次经过「列选择 → 批量删除 → **+ 新建用例** → **AI 用例生成**」（批量删除禁用时仍有焦点环）。Split 按钮获焦后 **Enter / Space / ↓** 展开下拉，**↑↓** 选菜单项，**Enter / Space** 执行后菜单自动收起。
