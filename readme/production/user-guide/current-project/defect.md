# 缺陷管理 [/project/defect](/project/defect)

缺陷是测试工作中记录问题的基础单元，每个产品需求、工作任务、软件 BUG，都统一叫做缺陷。

## 什么是缺陷

在 Cat2Bug 平台中，缺陷是一个广义的概念，包括：
- **软件 BUG** - 程序错误、功能异常
- **产品需求** - 新功能、功能改进
- **工作任务** - 待办事项、优化任务

通过统一的缺陷管理，可以：
- 跟踪问题的完整生命周期
- 协调开发和测试工作
- 评估产品质量
- 积累问题知识库

## 名词解释

| 名词 | 说明 |
|------|------|
| 缺陷 | 需求、任务、BUG 统称缺陷，是测试工作中记录问题的基础单元 |
| 缺陷状态 | 缺陷在生命周期中的当前阶段，如开启、修复中、待验证、已解决、已关闭等 |
| 存活时间 | 缺陷从最近一次开始时间算起，直到结束或至今的时间区间 |
| 驳回次数 | 缺陷从最近一次开始时间算起，直到结束或至今的时间区间内，反复被驳回的次数总和 |
| 处理人 | 当前负责处理缺陷的人员，通常是开发人员或测试人员 |

## 缺陷生命周期

缺陷的生命周期如下图所示：

![缺陷流程](./images/defect-flow.png)

**主要状态流转：**
1. **新建** - 测试人员创建缺陷
2. **开启** - 缺陷进入处理流程
3. **修复中** - 开发人员修复问题
4. **待验证** - 等待测试人员验证
5. **已解决** - 验证通过，问题解决
6. **已关闭** - 缺陷关闭
7. **驳回** - 验证未通过，返回修复

## 数据统计

系统提供了多种缺陷统计模版，点击缺陷列表右上角的【+】图标，跳转到缺陷模版页面，可选择显示的统计模版。

::: tip 提示
可通过拖动操作调整模版显示顺序
:::

![统计模版](./images/defect-statistics-option.png)

当设置完模版后，返回缺陷列表页面，即可显示所有设定的缺陷统计数据，如下图：

![带统计的缺陷列表](./images/defect-list-statistics.png)

::: tip 提示
多数统计面板中的参数，都可以与列表筛选联动，点击某个数据后，可按照点击数据进行筛选
:::

**统计模版类型：**

1. **缺陷趋势图** - 显示缺陷数量随时间的变化
2. **状态分布** - 显示各状态缺陷的数量
3. **优先级分布** - 显示各优先级缺陷的数量
4. **类型分布** - 显示各类型缺陷的数量
5. **处理人统计** - 显示各成员处理的缺陷数量
6. **模块分布** - 显示各交付物的缺陷数量

## 缺陷管理与其他模块的关系

### 与测试用例的关系

- 缺陷可以关联测试用例
- 记录在哪个测试步骤发现的问题
- 用例执行失败时可直接创建缺陷

### 与测试计划的关系

- 执行测试计划时发现问题创建缺陷
- 缺陷修复后需要在测试计划中回归验证

### 与交付物的关系

- 缺陷必须关联到某个交付物
- 按交付物统计缺陷分布
- 评估各模块的质量状况

## 常见问题

### Q: 如何批量操作缺陷？

A: 在缺陷列表中勾选多个缺陷，然后点击批量操作按钮（如批量指派、批量关闭等）。

### Q: 缺陷可以关联多个测试用例吗？

A: 目前一个缺陷只能关联一个测试用例。如果需要关联多个，可以在缺陷描述中添加其他用例的链接。

### Q: 如何查看我负责的缺陷？

A: 可以使用页标签功能，创建一个"我负责的"标签，设置筛选条件为"处理人=我"。

### Q: 缺陷可以导出吗？

A: 可以。在缺陷列表页面点击【导出】按钮，可以导出为 Excel 格式。

### Q: 如何查看缺陷的历史记录？

A: 打开缺陷详情页，可以看到缺陷的所有操作历史，包括创建、修改、状态变更等。
