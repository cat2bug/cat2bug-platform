# 删除缺陷

删除操作为**软删除**：缺陷数据仍保留在系统中，可在「已删除缺陷」页签中查看，并支持**恢复**。删除与恢复操作会写入缺陷日志（`sys_defect_log`），无需单独记录删除时间字段。

## 使用场景

- 创建错误的缺陷
- 测试数据需要清理
- 确认无效的缺陷记录

## 操作步骤

### 1. 点击删除

在缺陷列表或缺陷详情中点击「删除」按钮。

### 2. 确认删除

在弹出的确认对话框中，确认删除操作。

### 3. 查看已删除缺陷

在缺陷列表页签栏中，切换到「全部缺陷」右侧的「已删除缺陷」页签，可查看本项目内所有已软删除的缺陷。

### 4. 恢复缺陷

在「已删除缺陷」列表的操作列中，点击「查看」可只读浏览详情；点击「恢复」可将缺陷还原到正常列表（「全部缺陷」及未勾选「已删除」的自定义页签）。

## 删除影响

软删除缺陷会导致：

- 缺陷在「全部缺陷」及默认自定义页签中不再显示
- 缺陷详情、工作流、编辑、Excel 内联编辑等变更操作被禁止（已删除 Tab 下表格仅可查看与恢复）
- 统计与计划等汇总默认不再计入已删除缺陷
- 删除与恢复操作记录在缺陷日志中

::: tip 提示
1. 软删除可恢复，误删请到「已删除缺陷」页签操作恢复
2. 只有缺陷创建人、测试人员、项目管理员有删除与恢复权限（与删除权限一致）
3. 对于已处理的缺陷，建议使用「关闭」而不是「删除」，以保留活跃列表中的历史记录
4. 自定义页签可在新建时通过底部「已删除」开关，筛选仅已删除或仅未删除的缺陷
:::
