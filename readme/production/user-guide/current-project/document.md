# 文档管理 [/project/document](/project/document)

文档模块用于备份项目相关文件，目前可通过创建、更新、删除进行管理。

## 什么是文档管理

在项目开发和测试过程中，会产生大量的文档资料，如需求文档、设计文档、测试方案、会议纪要等。文档管理模块提供了一个集中存储和管理这些文档的地方。

**文档管理的作用：**

- 集中存储项目文档
- 方便团队成员查阅
- 保留文档历史版本
- 支持文档分类管理
- 便于文档检索

## 文档列表

文档列表展示了项目中的所有文档，支持多种查看和操作方式。

![文档列表](../../../images/user-guide/current-project/document/doc-list.png)

### 列表信息

每个文档显示以下信息：

- **文档名称** - 文档的标题
- **文档类型** - 需求文档、设计文档、测试文档等
- **文件大小** - 文档的大小
- **创建人** - 文档上传者
- **创建时间** - 文档上传时间
- **操作按钮** - 查看、下载、编辑、删除等操作

### 筛选功能

通过筛选条件快速找到需要的文档。

**按文档类型筛选：**
- 需求文档 - 产品需求、功能需求
- 设计文档 - 架构设计、详细设计
- 测试文档 - 测试方案、测试报告
- 会议纪要 - 会议记录、决策记录
- 技术文档 - API 文档、部署文档
- 其他文档 - 其他类型的文档

**按创建人筛选：**
选择创建人，查看指定成员上传的文档。

**按时间筛选：**
- 今天 - 今天上传的文档
- 本周 - 本周上传的文档
- 本月 - 本月上传的文档
- 自定义时间 - 指定时间范围

**按标签筛选：**
选择标签，查看带有指定标签的文档。

### 搜索功能

在搜索框中输入关键字，快速搜索文档。

**搜索范围：**
- 文档标题
- 文档描述
- 文档内容（支持全文搜索）

**搜索特点：**
- 支持模糊匹配
- 实时搜索
- 高亮显示匹配内容
- 搜索结果排序

### 排序功能

点击列表表头，可以按不同字段排序。

**支持的排序字段：**
- 文档名称 - 按名称字母顺序排序
- 创建时间 - 按上传时间排序
- 更新时间 - 按最后修改时间排序
- 文件大小 - 按文件大小排序

**排序方式：**
- 升序 - 从小到大、从旧到新、A-Z
- 降序 - 从大到小、从新到旧、Z-A

## 文档类型

### 上传文档

支持上传多种格式的文档文件。

**支持的文件格式：**
- **文档类** - Word (.doc, .docx), PDF (.pdf), Excel (.xls, .xlsx), PPT (.ppt, .pptx)
- **文本类** - TXT (.txt), Markdown (.md)
- **图片类** - JPG, PNG, GIF, BMP
- **压缩包** - ZIP, RAR, 7Z
- **其他** - 根据系统配置支持更多格式

**文件大小限制：** 默认单个文件最大 500MB

### 在线创建文档

支持在线创建 Markdown 格式的文档。

**Markdown 编辑器功能：**
- 实时预览
- 语法高亮
- 快捷工具栏
- 支持插入图片、表格、代码块
- 支持导出为 HTML、PDF

## 文档版本管理

### 版本历史

系统会自动保存文档的历史版本，可以查看和恢复历史版本。

**查看版本历史：**
1. 打开文档详情页
2. 点击【版本历史】标签
3. 查看所有历史版本
4. 每个版本显示：
   - 版本号
   - 更新时间
   - 更新人
   - 更新说明

### 版本对比

可以对比不同版本的文档内容，查看变更内容。

**对比步骤：**
1. 在版本历史中选择两个版本
2. 点击【对比】按钮
3. 查看差异内容
4. 支持高亮显示变更部分

### 版本恢复

可以恢复到历史版本。

**恢复步骤：**
1. 在版本历史中找到要恢复的版本
2. 点击【恢复】按钮
3. 确认恢复操作
4. 系统会创建一个新版本，内容为历史版本的内容

## 最佳实践

### 文档命名建议

1. **使用清晰的名称**
   - 包含文档类型
   - 包含版本号
   - 包含日期

2. **命名示例**
   ```
   好的命名：
   - 用户管理模块需求文档_v1.0_20240101
   - 系统架构设计_v2.0_20240115
   - 测试报告_第一轮_20240120

   不好的命名：
   - 文档1
   - 需求
   - test
   ```

### 文档组织建议

1. **按类型分类**
   - 使用文档类型字段
   - 使用标签辅助分类
   - 定期整理归档

2. **版本管理**
   - 重要变更时更新版本
   - 填写详细的更新说明
   - 保留关键版本

3. **权限控制**
   - 敏感文档限制访问
   - 定期 review 权限设置
   - 及时回收离职人员权限

### 文档维护建议

1. **定期清理**
   - 删除过期文档
   - 归档历史文档
   - 合并重复文档

2. **保持更新**
   - 及时更新文档内容
   - 同步最新的需求变更
   - 补充遗漏的文档

3. **质量控制**
   - 文档格式统一
   - 内容准确完整
   - 定期 review 文档质量

## 常见问题

### Q: 支持哪些文件格式？

A: 支持常见的文档格式（Word、PDF、Excel、PPT）、文本格式（TXT、Markdown）、图片格式（JPG、PNG）、压缩包（ZIP、RAR）等。具体支持的格式可以在系统设置中查看。

### Q: 文件大小有限制吗？

A: 有限制。默认单个文件最大 500MB。如果需要上传更大的文件，请联系系统管理员调整配置。

### Q: 如何批量上传文档？

A: 可以将多个文件打包成 ZIP 文件，然后上传 ZIP 文件。系统会自动解压并创建多个文档。

### Q: 文档可以分享给外部人员吗？

A: 目前外部人员角色不能查看文档。如果需要分享，可以下载文档后通过其他方式（邮件、网盘等）分享。

### Q: 如何恢复误删的文档？

A: 删除操作不可恢复。建议在删除前确认文档确实不再需要。重要文档建议定期备份到本地或其他存储系统。

### Q: 文档可以关联到缺陷或测试用例吗？

A: 目前不支持直接关联。但可以在缺陷或测试用例的描述中添加文档链接，或者使用相同的标签来建立关联关系。
