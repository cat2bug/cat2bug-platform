# 编辑文档

编辑文档的信息或内容，更新文档到最新状态。

## 使用场景

- 修改文档标题或描述
- 更新文档内容
- 调整文档分类
- 修改文档标签
- 更新文档版本

## 操作步骤

### 1. 找到要编辑的文档

在文档列表中找到要编辑的文档。

### 2. 点击编辑按钮

点击文档右侧的【编辑】按钮，打开编辑页面。

![编辑文档](../../../images/user-guide/current-project/document/document-edit.png)

### 3. 编辑文档信息

#### 修改文档标题

更新文档的标题。

**注意事项：**
- 标题不能为空
- 建议使用清晰的名称
- 修改标题不影响文档内容

#### 修改文档描述

更新文档的详细描述。

**描述内容建议：**
- 说明文档的主要内容
- 记录重要的更新信息
- 标注文档的适用范围

#### 修改文档类型

更改文档的类型分类。

**文档类型：**
- 需求文档
- 设计文档
- 测试文档
- 会议纪要
- 技术文档
- 其他文档

#### 修改文档标签

添加、删除或修改文档标签。

**标签操作：**
- 点击标签输入框
- 选择已有标签或创建新标签
- 点击标签的 × 删除标签
- 一个文档可以有多个标签

### 4. 编辑文档内容

对于 Markdown 文档，可以直接编辑内容。

#### 使用编辑器

**编辑器功能：**
- 左侧编辑区
- 右侧实时预览
- 工具栏快捷操作
- 快捷键支持

**编辑操作：**
- 修改文本内容
- 插入图片
- 添加表格
- 插入代码块
- 调整格式

#### 版本说明

编辑文档时，建议填写版本说明。

**版本说明内容：**
- 本次修改的内容
- 修改的原因
- 影响范围
- 注意事项

**示例：**
```
更新内容：
1. 修正了接口参数说明
2. 添加了错误码说明
3. 更新了示例代码
```

### 5. 保存修改

点击【保存】按钮保存修改。

**保存选项：**
- **保存** - 保存并继续编辑
- **保存并关闭** - 保存并返回列表
- **取消** - 放弃修改

## 更新文档文件

对于上传的文档文件，可以上传新版本替换。

### 更新步骤

#### 1. 点击更新按钮

在文档详情页面，点击【更新】按钮。

#### 2. 上传新文件

选择新版本的文档文件。

**更新规则：**
- 文件格式必须与原文件相同
- 文件大小不能超过限制
- 系统会保留历史版本

#### 3. 填写更新说明

说明本次更新的内容。

**更新说明建议：**
- 详细说明更新内容
- 标注重要变更
- 说明更新原因

#### 4. 确认更新

点击【确定】完成更新。

**更新结果：**
- 文档版本号自动增加
- 保留历史版本
- 更新时间更新为当前时间
- 记录更新人和更新说明

## 批量编辑

可以批量修改多个文档的某些属性。

### 批量修改类型

1. 勾选要修改的多个文档
2. 点击【批量编辑】按钮
3. 选择【修改类型】
4. 选择新的文档类型
5. 点击【确定】完成修改

### 批量修改标签

1. 勾选要修改的多个文档
2. 点击【批量编辑】按钮
3. 选择【修改标签】
4. 添加或删除标签
5. 点击【确定】完成修改

**批量修改规则：**
- 只修改有权限编辑的文档
- 跳过没有权限的文档
- 显示修改结果统计

## 编辑权限

只有以下角色有编辑权限：

### 可以编辑的角色

- **项目创建人** - 可以编辑所有文档
- **项目管理员** - 可以编辑所有文档
- **文档创建人** - 可以编辑自己创建的文档

### 权限限制

- 普通成员不能编辑他人创建的文档
- 外部人员不能编辑文档
- 编辑操作会记录操作日志

## 编辑历史

系统会记录文档的编辑历史。

### 查看编辑历史

1. 打开文档详情页面
2. 点击【版本历史】标签
3. 查看所有编辑记录

**历史记录包含：**
- 编辑时间
- 编辑人
- 版本号
- 更新说明
- 文件大小

### 对比版本

可以对比不同版本的差异。

**对比步骤：**
1. 在版本历史中选择两个版本
2. 点击【对比】按钮
3. 查看差异内容
4. 高亮显示变更部分

### 恢复版本

可以恢复到历史版本。

**恢复步骤：**
1. 在版本历史中找到要恢复的版本
2. 点击【恢复】按钮
3. 确认恢复操作
4. 系统创建新版本，内容为历史版本的内容

## 编辑冲突处理

当多人同时编辑同一文档时，可能会产生冲突。

### 冲突检测

**系统会检测：**
- 是否有其他人正在编辑
- 文档是否已被修改
- 版本是否一致

**冲突提示：**
- 显示正在编辑的用户
- 提示文档已被修改
- 建议刷新后再编辑

### 冲突解决

**方式一：放弃修改**
1. 关闭编辑页面
2. 刷新文档
3. 查看最新版本
4. 重新编辑

**方式二：强制保存**
1. 复制自己的修改内容
2. 刷新文档查看最新版本
3. 合并两个版本的内容
4. 保存合并后的内容

**方式三：另存为新文档**
1. 将修改内容另存为新文档
2. 保留两个版本
3. 后续再合并

## 自动保存

编辑器支持自动保存功能。

**自动保存规则：**
- 每 30 秒自动保存一次
- 内容有变化时才保存
- 保存为草稿状态
- 不影响正式版本

**保存状态：**
- "已保存" - 内容已保存
- "保存中..." - 正在保存
- "未保存" - 有未保存的更改

## 最佳实践

### 编辑前准备

1. **确认编辑权限**
   - 检查是否有编辑权限
   - 确认没有其他人正在编辑
   - 查看最新版本

2. **备份重要内容**
   - 重要文档先下载备份
   - 复制关键内容
   - 记录当前版本号

3. **规划修改内容**
   - 明确要修改的内容
   - 准备好更新说明
   - 确认修改的影响范围

### 编辑时注意

1. **填写版本说明**
   - 详细说明修改内容
   - 标注重要变更
   - 便于后续追溯

2. **及时保存**
   - 定期手动保存
   - 利用自动保存功能
   - 避免内容丢失

3. **预览效果**
   - 使用实时预览功能
   - 检查格式是否正确
   - 确认内容完整

### 编辑后检查

1. **验证修改结果**
   - 检查修改是否生效
   - 验证内容是否正确
   - 确认格式是否正常

2. **通知相关人员**
   - 通知文档使用者
   - 说明更新内容
   - 提醒查看新版本

3. **更新相关文档**
   - 更新关联文档
   - 保持文档一致性
   - 避免信息不同步

::: tip 提示
1. 编辑文档前建议先查看是否有其他人正在编辑
2. 重要修改建议填写详细的版本说明
3. 编辑器支持自动保存，防止内容丢失
4. 可以查看版本历史和对比差异
5. 批量编辑可以提高效率，但要确保修改的合理性
:::
