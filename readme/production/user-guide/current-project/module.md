# 交付物管理 [/project/module](/project/module)

交付物是项目中最终交付给客户的目标或成果，在软件中，代表某个系统、手机 APP、微信小程序。

多个交付物可以以父子上下级的结构体现，如某某系统下可以关联多个模块。在 Cat2Bug-Platform 中，交付物会关联到[测试用例](case.md)和[缺陷](defect.md)中。

## 什么是交付物

在 Cat2Bug-Platform 中，交付物是组织测试工作的重要维度，它代表了项目中需要测试的具体对象，是需要交付的软/硬件统称。

**交付物的作用：**

- 组织测试用例的结构
- 关联缺陷到具体模块
- 统计各模块的测试情况
- 追踪各模块的质量状态

## 交付物结构

多个交付物可以以父子上下级的结构体现，如某某系统下可以关联多个模块。

**示例结构：**

```
电商系统
├── 用户模块
│   ├── 用户注册
│   ├── 用户登录
│   └── 个人中心
├── 商品模块
│   ├── 商品列表
│   ├── 商品详情
│   └── 商品搜索
└── 订单模块
    ├── 购物车
    ├── 下单
    └── 订单管理
```

## 交付物列表

交付物列表展示了项目中的所有交付物，以树形结构呈现交付物的层级关系。

![交付物列表](../../../images/user-guide/current-project/module/deliverable-list.png)

### 列表信息

每个交付物显示以下信息：

- **交付物名称** - 交付物的名称
- **附件** - 交付物的附件数量
- **备注** - 交付物的备注信息
- **操作按钮** - 编辑、删除等操作

### 树形结构操作

**展开/折叠：**
- 点击节点前的 ▷ 图标展开子节点
- 点击节点前的 ▽ 图标折叠子节点
- 点击工具栏的【全部展开】按钮展开所有节点
- 点击工具栏的【全部折叠】按钮折叠所有节点

### 搜索功能

在搜索框中输入关键字，可以快速找到交付物。

**搜索范围：**
- 交付物名称

**搜索特点：**
- 支持模糊匹配
- 实时搜索
- 高亮显示匹配内容
- 搜索时会自动展开包含匹配结果的节点

## 交付物与其他模块的关系

### 与测试用例的关系

- 每个测试用例必须关联一个交付物
- 通过交付物可以筛选相关的测试用例
- 交付物的层级结构帮助组织用例

### 与缺陷的关系

- 每个缺陷必须关联一个交付物
- 通过交付物可以统计各模块的缺陷情况
- 便于定位问题集中的模块

### 与项目的关系

- 交付物属于某个项目
- 一个项目可以有多个交付物
- 交付物的结构反映了项目的架构

## 最佳实践

### 交付物命名建议

1. **使用清晰的名称**
   - 避免使用缩写
   - 使用业务术语
   - 保持命名一致性

2. **合理的层级深度**
   - 建议不超过 4 层
   - 每层的粒度要合适
   - 避免过度细分

3. **示例命名**
   ```
   好的命名：
   - 用户管理模块
   - 商品搜索功能
   - 订单支付流程

   不好的命名：
   - 模块1
   - test
   - 功能A
   ```

### 交付物组织建议

1. **按功能模块划分**
   ```
   系统
   ├── 用户模块
   ├── 商品模块
   ├── 订单模块
   └── 支付模块
   ```

2. **按系统层次划分**
   ```
   系统
   ├── 前端
   │   ├── Web端
   │   └── 移动端
   └── 后端
       ├── API服务
       └── 数据库
   ```

3. **按业务流程划分**
   ```
   系统
   ├── 注册流程
   ├── 登录流程
   ├── 购物流程
   └── 支付流程
   ```

### 维护建议

1. **定期 review**
   - 检查交付物结构是否合理
   - 删除不再使用的交付物
   - 合并重复的交付物

2. **保持同步**
   - 交付物结构应与系统架构保持一致
   - 系统架构变更时及时更新交付物
   - 新增功能时及时创建对应交付物

3. **统一管理**
   - 指定专人负责交付物的维护
   - 制定交付物命名规范
   - 定期培训团队成员

## 常见问题

### Q: 交付物和模块有什么区别？

A: 在 Cat2Bug-Platform 中，交付物就是模块的概念，两者是同一个东西。我们使用"交付物"这个术语是为了更贴近项目管理的语境。

### Q: 可以修改交付物的父级吗？

A: 可以。通过拖拽方式可以调整交付物的层级关系，或者在编辑时重新选择父级交付物。

### Q: 删除交付物会影响已有的测试用例和缺陷吗？

A: 如果交付物下有关联的测试用例或缺陷，系统会阻止删除操作。需要先处理这些关联数据（删除或转移到其他交付物）才能删除交付物。

### Q: 如何快速找到某个交付物？

A: 可以使用搜索功能，输入交付物名称的关键字进行搜索。也可以通过展开/折叠树形结构来浏览。

### Q: 交付物的层级有限制吗？

A: 理论上没有层级限制，但建议不超过 4 层，以保持结构清晰。层级过深会增加管理复杂度。
