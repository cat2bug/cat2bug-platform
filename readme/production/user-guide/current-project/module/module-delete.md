# 删除交付物

删除不再需要的交付物。

## 使用场景

- 删除创建错误的交付物
- 清理废弃的交付物
- 项目架构调整时删除旧交付物
- 测试数据清理

## 删除条件

删除交付物前需要确认以下条件：

- ✅ 该交付物下没有关联的测试用例
- ✅ 该交付物下没有关联的缺陷
- ✅ 该交付物下没有子交付物

如果不满足以上条件，系统会阻止删除操作。

## 操作步骤

### 1. 找到要删除的交付物

在交付物列表中找到要删除的交付物。

### 2. 点击删除按钮

点击交付物右侧的【删除】按钮。

![删除交付物](../../../images/deliverable-delete.png)

### 3. 确认删除

在弹出的确认对话框中，确认删除操作。

### 4. 完成删除

系统删除交付物并刷新列表。

## 删除前的准备

如果交付物下有关联数据，需要先处理这些数据才能删除。

### 处理关联的测试用例

**方式一：删除用例**

1. 进入测试用例列表
2. 筛选该交付物的用例
3. 批量删除用例

**方式二：转移用例**

1. 进入测试用例列表
2. 筛选该交付物的用例
3. 批量修改用例的交付物
4. 选择新的交付物

### 处理关联的缺陷

**方式一：删除缺陷**

1. 进入缺陷列表
2. 筛选该交付物的缺陷
3. 批量删除缺陷

**方式二：转移缺陷**

1. 进入缺陷列表
2. 筛选该交付物的缺陷
3. 批量修改缺陷的交付物
4. 选择新的交付物

### 处理子交付物

**方式一：删除子交付物**

1. 先删除子交付物（需要先处理子交付物的关联数据）
2. 再删除父交付物

**方式二：调整子交付物**

1. 将子交付物移动到其他父级下
2. 或将子交付物设为根节点
3. 再删除原父交付物

## 批量删除

如果需要删除多个交付物，可以使用批量删除功能。

### 批量删除步骤

1. 勾选要删除的多个交付物
2. 点击【批量删除】按钮
3. 确认删除操作
4. 系统检查删除条件
5. 满足条件的交付物被删除
6. 不满足条件的交付物保留并提示原因

### 批量删除规则

- 只删除满足删除条件的交付物
- 有关联数据的交付物不会被删除
- 有子交付物的交付物不会被删除
- 删除结果会显示成功和失败的数量

## 删除影响

删除交付物会导致：

- ❌ 交付物信息永久删除，无法恢复
- ❌ 交付物的统计数据清除
- ❌ 交付物的历史记录删除

**不会影响：**
- ✅ 已转移到其他交付物的用例和缺陷
- ✅ 其他交付物的数据
- ✅ 项目的其他数据

## 权限说明

只有以下角色有删除权限：

- **项目创建人** - 可以删除所有交付物
- **项目管理员** - 可以删除所有交付物
- **交付物创建人** - 可以删除自己创建的交付物

::: tip 提示
1. 删除操作不可恢复，请谨慎操作
2. 建议先转移关联数据，再删除交付物
3. 对于暂时不用的交付物，建议保留而不是删除
4. 删除父交付物前，必须先处理所有子交付物
5. 批量删除时，系统会自动跳过不满足条件的交付物
:::
