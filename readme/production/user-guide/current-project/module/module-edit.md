# 修改交付物

修改交付物的基本信息或调整交付物的层级关系。

## 使用场景

- 更新交付物名称或描述
- 调整交付物的负责人
- 修改交付物类型
- 调整交付物的层级关系
- 补充交付物信息

## 操作步骤

### 1. 找到要修改的交付物

在交付物列表中找到要修改的交付物。

### 2. 点击编辑按钮

点击交付物右侧的【编辑】按钮，打开编辑对话框。

![编辑交付物](../../../images/user-guide/current-project/module/deliverable-edit.png)

### 3. 修改交付物信息

#### 修改交付物名称

更新交付物的名称。

**注意事项：**
- 名称在同一父级下不能重复
- 修改名称不影响已关联的用例和缺陷
- 建议保持命名规范的一致性

#### 修改交付物描述

更新交付物的详细描述。

**建议补充的内容：**
- 功能范围的变更
- 技术实现的更新
- 业务逻辑的调整
- 依赖关系的说明

#### 修改父级交付物

调整交付物的层级关系。

**调整方式：**
- 选择新的父级交付物
- 选择"无"设为根节点
- 不能选择自己或自己的子节点作为父级

#### 修改负责人

更换交付物的负责人。

**更换场景：**
- 人员调整
- 职责变更
- 工作交接
- 负责人离职

#### 修改交付物类型

更新交付物的类型。

**类型选择：**
- 系统
- 模块
- 功能
- 组件

### 4. 保存修改

点击【确定】按钮保存修改。

## 拖拽调整层级

除了通过编辑对话框修改父级，还可以通过拖拽方式快速调整层级。

### 拖拽步骤

1. 在交付物树中选中要移动的交付物
2. 按住鼠标左键拖拽
3. 移动到目标位置
4. 释放鼠标完成移动
5. 系统自动保存新的层级关系

### 拖拽规则

- **可以拖拽到其他节点下** - 成为其子节点
- **可以拖拽到根节点** - 成为顶级节点
- **不能拖拽到自己的子节点下** - 避免循环引用
- **子节点随父节点移动** - 保持子树完整

### 拖拽提示

拖拽时会显示目标位置的提示线：

- **蓝色提示线** - 可以放置
- **红色提示线** - 不能放置（循环引用）

## 批量修改

如果需要批量修改多个交付物的某些属性，可以使用批量修改功能。

### 批量修改负责人

1. 勾选要修改的多个交付物
2. 点击【批量修改】按钮
3. 选择【修改负责人】
4. 选择新的负责人
5. 点击【确定】完成修改

### 批量调整父级

1. 勾选要移动的多个交付物
2. 点击【批量修改】按钮
3. 选择【调整父级】
4. 选择新的父级交付物
5. 点击【确定】完成调整

::: tip 提示
1. 修改交付物名称不会影响已关联的用例和缺陷
2. 调整层级关系时，子交付物会随父交付物一起移动
3. 修改负责人不会影响已有的任务分配
4. 拖拽调整层级比编辑对话框更快捷
5. 批量修改可以提高效率，但要确保修改的合理性
:::
