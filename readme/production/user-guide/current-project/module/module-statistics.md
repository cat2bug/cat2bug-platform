# 交付物统计

查看交付物的测试用例和缺陷统计数据，评估交付物的质量状况。

## 使用场景

- 了解各交付物的测试覆盖情况
- 分析各交付物的缺陷分布
- 识别高风险模块
- 评估交付物质量
- 制定测试计划

## 统计维度

### 用例统计

查看每个交付物关联的测试用例数量和分布。

#### 总用例数

显示该交付物及其所有子交付物的用例总数。

**统计规则：**
- 包含直接关联的用例
- 包含所有子交付物的用例
- 实时更新

#### 各级别用例分布

按用例级别统计用例数量：

- **P0** - 核心功能用例
- **P1** - 重要功能用例
- **P2** - 一般功能用例
- **P3** - 次要功能用例

**分布图表：**
- 饼图展示各级别占比
- 柱状图展示各级别数量
- 趋势图展示历史变化

#### 用例覆盖率

评估交付物的测试覆盖程度。

**计算方式：**
```
用例覆盖率 = (已创建用例数 / 预计用例数) × 100%
```

**覆盖率评级：**
- **优秀** - 覆盖率 ≥ 90%
- **良好** - 覆盖率 70% ~ 90%
- **一般** - 覆盖率 50% ~ 70%
- **较差** - 覆盖率 < 50%

### 缺陷统计

查看每个交付物的缺陷情况和分布。

#### 总缺陷数

显示该交付物及其所有子交付物的缺陷总数。

**统计规则：**
- 包含直接关联的缺陷
- 包含所有子交付物的缺陷
- 实时更新

#### 各状态缺陷分布

按缺陷状态统计缺陷数量：

- **处理中** - 开发正在修复的缺陷
- **待验证** - 等待测试验证的缺陷
- **已驳回** - 验证未通过的缺陷
- **已关闭** - 已验证通过的缺陷

**状态分布图表：**
- 饼图展示各状态占比
- 柱状图展示各状态数量
- 趋势图展示状态变化

#### 各优先级缺陷分布

按缺陷优先级统计缺陷数量：

- **急** - 系统崩溃、数据丢失、安全漏洞
- **高** - 核心功能无法使用
- **中** - 一般功能问题
- **低** - 界面优化、文案错误

**优先级分布图表：**
- 饼图展示各优先级占比
- 柱状图展示各优先级数量
- 热力图展示风险分布

#### 缺陷密度

评估交付物的缺陷集中程度。

**计算方式：**
```
缺陷密度 = 缺陷总数 / 用例总数
```

**密度评级：**
- **优秀** - 密度 < 0.1
- **良好** - 密度 0.1 ~ 0.3
- **一般** - 密度 0.3 ~ 0.5
- **较差** - 密度 > 0.5

## 质量评估

根据统计数据综合评估交付物的质量状况。

### 评估维度

#### 用例覆盖度

- 用例数量是否充足
- 各级别用例分布是否合理
- 核心功能是否有足够的用例

#### 缺陷严重度

- 高优先级缺陷数量
- 缺陷修复进度
- 遗留缺陷风险

#### 测试完成度

- 用例执行率
- 缺陷关闭率
- 测试进度

### 质量等级

综合评估后给出质量等级：

**优秀** ⭐⭐⭐⭐⭐
- 用例覆盖充分
- 缺陷少且优先级低
- 测试完成度高

**良好** ⭐⭐⭐⭐
- 用例覆盖较好
- 缺陷在可控范围内
- 测试基本完成

**一般** ⭐⭐⭐
- 用例覆盖一般
- 存在一些中等优先级缺陷
- 测试进行中

**较差** ⭐⭐
- 用例覆盖不足
- 存在较多高优先级缺陷
- 测试进度滞后

**很差** ⭐
- 用例严重不足
- 存在大量高优先级缺陷
- 测试未开始或刚开始

## 统计报表

### 交付物对比报表

对比多个交付物的统计数据：

**对比维度：**
- 用例数量对比
- 缺陷数量对比
- 质量等级对比
- 测试进度对比

**报表形式：**
- 对比表格
- 对比柱状图
- 雷达图

### 趋势分析报表

分析交付物的历史趋势：

**趋势维度：**
- 用例增长趋势
- 缺陷变化趋势
- 质量改善趋势
- 测试效率趋势

**报表形式：**
- 折线图
- 面积图
- 趋势预测

### 风险识别报表

识别高风险交付物：

**风险指标：**
- 高优先级缺陷数量
- 缺陷密度
- 用例覆盖率
- 测试进度

**风险等级：**
- 🔴 高风险 - 需要立即关注
- 🟡 中风险 - 需要重点测试
- 🟢 低风险 - 正常跟进

## 统计数据导出

可以将统计数据导出为 Excel 或 PDF 格式。

### 导出内容

- 交付物基本信息
- 用例统计数据
- 缺陷统计数据
- 质量评估结果
- 统计图表

### 导出步骤

1. 在统计页面点击【导出】按钮
2. 选择导出格式（Excel / PDF）
3. 选择导出内容
4. 点击【确定】生成文件
5. 下载导出文件

## 最佳实践

### 定期查看统计

建议每周查看一次交付物统计：

- 了解测试进度
- 发现问题趋势
- 调整测试策略
- 评估发布风险

### 关注关键指标

重点关注以下指标：

- **高优先级缺陷数** - 影响发布决策
- **缺陷密度** - 反映质量状况
- **用例覆盖率** - 评估测试充分性
- **缺陷关闭率** - 跟踪修复进度

### 横向对比分析

对比不同交付物的统计数据：

- 识别质量差异
- 发现共性问题
- 优化资源分配
- 总结经验教训

::: tip 提示
1. 统计数据包含交付物及其所有子交付物的数据
2. 统计数据实时更新，无需手动刷新
3. 可以点击统计数字快速跳转到详细列表
4. 建议结合测试计划和缺陷趋势综合分析
5. 导出的统计报表可用于项目汇报和质量评审
:::
