# 测试计划 [/project/plan](/project/plan)

测试计划是组织和管理测试活动的重要工具，帮助团队系统化地进行测试工作。

## 什么是测试计划

测试计划是对测试活动的整体规划，包括测试范围、测试策略、测试资源、测试进度等内容。通过测试计划，可以：

- 明确测试目标和范围
- 合理分配测试资源
- 制定测试时间表
- 跟踪测试进度
- 评估测试质量

## 名词解释

| 名词 | 说明 |
|------|------|
| 测试计划 | 对一次测试活动的完整规划，包含测试目标、测试范围、测试用例集、执行人员、时间安排等要素 |
| 测试用例 | 测试计划的基本执行单元，描述了具体的测试步骤和预期结果 |
| 执行状态 | 测试用例在测试计划中的执行状态，包括未执行、执行中、通过、失败、阻塞、跳过 |
| 通过率 | 通过用例数 / 已执行用例数 × 100%，是衡量测试质量的重要指标 |
| 用例覆盖率 | 已执行用例数 / 总用例数 × 100%，反映测试计划的执行进度 |

## 测试计划与其他模块的关系

### 与测试用例的关系

- 测试计划引用测试用例
- 一个用例可以被多个计划引用
- 用例的修改会影响所有引用它的计划

### 与缺陷的关系

- 执行测试计划时发现的问题会创建缺陷
- 缺陷会关联到对应的测试用例

### 与项目的关系

- 测试计划属于某个项目
- 一个项目可以有多个测试计划
- 项目成员可以查看和执行测试计划

## 键盘快捷键

通用说明见 [键盘快捷键](../../advanced/keyboard-shortcuts.md)。

在测试计划列表页（无弹框/抽屉/下拉时）可用 **空格** 打开动作面板，或按住 **⌘/Ctrl** 查看工具栏徽标：

| 字母 | 动作 |
|------|------|
| S | 聚焦查询（**← / →** 在计划名称、版本间切换；版本 **→** 进入工具栏首按钮） |
| E | 新建计划 |
| B | 上一页 |
| P | 下一页 |

按住 **⌘/Ctrl** 时，**当前可见行**在表格第一个可见列显示动态字母（优先 `1`–`9`）：列内有文字时徽标在**文字右下角**，内容为空时在**单元格居中**；按字母打开该计划执行（需执行权限）。

**显示字段**：工具栏首按钮（折叠图标）打开列选择浮层；浮层内 **↑** / **↓** 切换字段项，**↑** 在第一项时关闭浮层，**Enter** / **空格** 切换勾选。详见 [键盘快捷键](../../advanced/keyboard-shortcuts.md#工具栏按钮焦点显示字段等)。

**查询区**：焦点在查询条内时，**← / →** 在「计划名称 → 版本」间切换；版本按 **→** 进入右侧工具栏首按钮，该按钮按 **←** 回到版本。
