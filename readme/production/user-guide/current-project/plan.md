# 测试计划 [/project/plan](/project/plan)

测试计划是组织和管理测试活动的重要工具，帮助团队系统化地进行测试工作。

## 什么是测试计划

测试计划是对测试活动的整体规划，包括测试范围、测试策略、测试资源、测试进度等内容。通过测试计划，可以：

- 明确测试目标和范围
- 合理分配测试资源
- 制定测试时间表
- 跟踪测试进度
- 评估测试质量

## 测试计划与其他模块的关系

### 与测试用例的关系

- 测试计划引用测试用例
- 一个用例可以被多个计划引用
- 用例的修改会影响所有引用它的计划

### 与缺陷的关系

- 执行测试计划时发现的问题会创建缺陷
- 缺陷会关联到对应的测试用例
- 缺陷修复后需要在测试计划中回归验证

### 与项目的关系

- 测试计划属于某个项目
- 一个项目可以有多个测试计划
- 项目成员可以查看和执行测试计划

## 常见问题

### Q: 测试计划可以修改吗？

A: 可以。测试计划在执行过程中可以根据实际情况调整，如添加或删除用例、调整执行人、修改时间等。

### Q: 如何处理阻塞的用例？

A: 将用例状态标记为"阻塞"，并在备注中说明阻塞原因。待阻塞问题解决后，再继续执行。

### Q: 测试计划可以复用吗？

A: 可以。可以将已有的测试计划复制为新的计划，然后根据需要调整内容。

### Q: 如何评估测试是否充分？

A: 可以从以下几个方面评估：
- 用例覆盖率是否达标
- 核心功能是否充分测试
- 缺陷收敛趋势是否良好
- 是否还有未测试的风险点
