# 测试计划 [/project/plan](/project/plan)

测试计划是组织和管理测试活动的重要工具，帮助团队系统化地进行测试工作。

## 什么是测试计划

测试计划是对测试活动的整体规划，包括测试范围、测试策略、测试资源、测试进度等内容。通过测试计划，可以：

- 明确测试目标和范围
- 合理分配测试资源
- 制定测试时间表
- 跟踪测试进度
- 评估测试质量

## 名词解释

| 名词 | 说明 |
|------|------|
| 测试计划 | 对一次测试活动的完整规划，包含测试目标、测试范围、测试用例集、执行人员、时间安排等要素 |
| 测试用例 | 测试计划的基本执行单元，描述了具体的测试步骤和预期结果 |
| 执行状态 | 测试用例在测试计划中的执行状态，包括未执行、执行中、通过、失败、阻塞、跳过 |
| 通过率 | 通过用例数 / 已执行用例数 × 100%，是衡量测试质量的重要指标 |
| 用例覆盖率 | 已执行用例数 / 总用例数 × 100%，反映测试计划的执行进度 |

## 测试计划与其他模块的关系

### 与测试用例的关系

- 测试计划引用测试用例
- 一个用例可以被多个计划引用
- 用例的修改会影响所有引用它的计划

### 与缺陷的关系

- 执行测试计划时发现的问题会创建缺陷
- 缺陷会关联到对应的测试用例

### 与项目的关系

- 测试计划属于某个项目
- 一个项目可以有多个测试计划
- 项目成员可以查看和执行测试计划
