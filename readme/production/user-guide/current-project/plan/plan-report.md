# 测试报告

测试计划完成后，可以生成测试报告，全面展示测试结果和质量评估。

## 生成测试报告

在测试计划详情页，点击【生成报告】按钮，系统会自动生成测试报告。

![生成测试报告](../../../images/user-guide/current-project/plan/plan_generate_report.png)

## 报告内容

测试报告包含以下主要内容：

### 测试概述

展示测试的基本信息，包括：

- 测试目标
- 测试范围
- 测试时间
- 参与人员

![测试概述](../../../images/user-guide/current-project/plan/plan_report_overview.png)

### 测试结果

统计用例执行情况，包括：

- 总用例数
- 已执行用例数
- 通过用例数
- 失败用例数
- 阻塞用例数
- 通过率

![测试结果统计](../../../images/user-guide/current-project/plan/plan_report_result.png)

### 缺陷分析

分析测试过程中发现的缺陷，包括：

- 缺陷总数
- 按严重程度分布
- 按优先级分布
- 按状态分布
- 缺陷趋势图

![缺陷分析](../../../images/user-guide/current-project/plan/plan_report_defect.png)

### 测试结论

给出测试结论和建议：

- 测试是否通过
- 是否可以发布
- 存在的主要问题
- 改进建议

![测试结论](../../../images/user-guide/current-project/plan/plan_report_conclusion.png)

### 风险评估

评估当前存在的风险：

- 未测试的功能点
- 未修复的严重缺陷
- 测试覆盖不足的区域
- 其他潜在风险

![风险评估](../../../images/user-guide/current-project/plan/plan_report_risk.png)

## 报告导出

测试报告支持导出多种格式，方便分享和归档。

### Word 格式

适合打印和归档，保留完整的格式和图表。

![导出Word](../../../images/user-guide/current-project/plan/plan_export_word.png)

### PDF 格式

适合分享和展示，跨平台兼容性好。

![导出PDF](../../../images/user-guide/current-project/plan/plan_export_pdf.png)

### Excel 格式

适合数据分析，可以进一步处理统计数据。

![导出Excel](../../../images/user-guide/current-project/plan/plan_export_excel.png)

## 报告查看

生成的报告会保存在系统中，可以随时查看历史报告。

![报告列表](../../../images/user-guide/current-project/plan/plan_report_list.png)

## 使用建议

> **提示：**
> 1. 在测试计划完成后及时生成报告
> 2. 报告中的数据要准确，确保统计无误
> 3. 测试结论要明确，给出清晰的发布建议
> 4. 风险评估要全面，不遗漏重要问题
> 5. 定期导出报告进行归档，便于后续查阅
