# AI大模型 [/project/project-ai](/project/project-ai)

配置项目使用的AI大模型服务。系统集成了Ollama API，可以方便地管理和使用本地或远程的大语言模型。

## Ollama集成

Cat2Bug平台集成了Ollama API，支持通过接口维护和使用Ollama提供的各种开源大模型。

### 什么是Ollama

Ollama是一个轻量级的大模型运行框架，支持在本地运行多种开源大语言模型，如Llama、Mistral、Qwen等。通过Ollama，您可以：
- 在本地部署和运行大模型，保护数据隐私
- 无需依赖外部API服务，降低使用成本
- 支持多种开源模型，灵活选择

## 功能说明

**主要功能：**
- 配置Ollama服务地址
- 查看可用的AI模型列表
- 添加和删除AI模型
- 测试模型连接
- 设置默认使用的模型

**操作步骤：**

### 1. 配置Ollama服务
1. 点击"AI大模型"
2. 输入Ollama服务地址（如：http://localhost:11434）
3. 点击"测试连接"验证配置
4. 保存配置

### 2. 管理AI模型
1. 在模型列表中查看已安装的模型
2. 点击"添加模型"，输入模型名称（如：llama3、qwen2等）
3. 系统会通过Ollama API拉取模型
4. 等待模型下载完成
5. 选择默认使用的模型

### 3. 使用AI模型
配置完成后，可以在以下场景中使用AI功能：
- 点击"测试"按钮验证模型是否正常工作
- 在缺陷管理中使用AI辅助生成描述
- 在测试用例中使用AI生成测试场景

## 支持的模型

通过Ollama，系统支持以下类型的开源模型：

**推荐模型：**
- **Llama 3** - Meta开源的高性能模型
- **Qwen 2** - 阿里通义千问开源模型
- **Mistral** - 高效的开源模型
- **Gemma** - Google开源的轻量级模型
- **DeepSeek** - 深度求索开源模型

**模型选择建议：**
- 小型项目：推荐使用7B参数的模型（如llama3:7b）
- 中型项目：推荐使用13B-14B参数的模型
- 大型项目：推荐使用70B以上参数的模型（需要较高硬件配置）

## Ollama安装

如果您还没有安装Ollama，请访问：
- 官方网站：https://ollama.ai
- 安装文档：https://github.com/ollama/ollama

**快速安装：**
```bash
# macOS/Linux
curl -fsSL https://ollama.ai/install.sh | sh

# Windows
下载安装包：https://ollama.ai/download
```

**启动Ollama服务：**
```bash
ollama serve
```

**拉取模型：**
```bash
# 拉取Llama 3模型
ollama pull llama3

# 拉取Qwen 2模型
ollama pull qwen2
```

## 常见问题

**Q: 为什么选择Ollama而不是OpenAI？**  
A: Ollama支持本地部署，数据不会发送到外部服务器，更好地保护项目隐私。同时无需支付API费用，适合长期使用。

**Q: Ollama需要什么硬件配置？**  
A: 建议至少8GB内存，推荐16GB以上。GPU可以显著提升推理速度，但不是必需的。

**Q: 可以同时配置多个AI模型吗？**  
A: 可以。系统支持配置多个模型，可以根据不同场景选择使用。

**Q: AI模型连接测试失败怎么办？**  
A: 请检查Ollama服务是否正常运行、服务地址是否正确、网络连接是否正常、防火墙是否允许访问。

**Q: 使用本地AI模型会泄露项目数据吗？**  
A: 不会。使用Ollama本地部署的模型，所有数据都在本地处理，不会发送到外部服务器。

**Q: 模型下载很慢怎么办？**  
A: 可以使用国内镜像源，或者手动下载模型文件后导入Ollama。

**Q: 如何切换使用不同的模型？**  
A: 在模型列表中选择要使用的模型，点击"设为默认"即可。
