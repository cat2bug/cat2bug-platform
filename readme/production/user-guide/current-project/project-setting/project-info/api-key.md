# API KEY [/project/project-api](/project/project-api)

管理项目的API访问密钥，用于第三方系统集成和API调用。

## 功能说明

API KEY页面提供以下功能：

- **添加API** - 创建新的API密钥并配置权限
- **查看API列表** - 显示所有已创建的API密钥
- **修改API** - 修改API密钥的配置
- **删除API** - 删除不再使用的API密钥

## 添加API

### 操作步骤

1. 进入项目设置 → 项目信息 → API KEY
2. 点击【新增Key】按钮
3. 在弹出的对话框中填写以下信息：
   - **API别名** - 为API密钥设置一个易识别的名称
   - **启用状态** - 选择是否启用该API密钥
   - **有效时间** - 设置API密钥的过期时间（可选）
   - **备注** - 添加备注说明（可选，最多255字符）
   - **权限配置** - 勾选该API密钥可以访问的功能模块和操作权限
4. 点击【确定】按钮完成创建

![添加项目API](../../../../../images/user-guide/current-project/project-setting/project-setting-api-create.png)

### 权限配置说明

可以为API密钥配置以下模块的权限：

**缺陷管理**
- 列表、查询、创建、更新、修复、指派、驳回、通过、关闭、开启

**测试用例管理**
- 列表、查询、创建、更新、删除

**交付物管理**
- 列表、查询、创建

**文件管理**
- 上传

**项目信息**
- 信息、成员

## 管理API

### 查看API列表

API列表显示以下信息：
- **API别名** - API密钥的名称
- **API Key** - API密钥内容，可以点击眼睛图标显示/隐藏，点击复制图标复制密钥
- **启用状态** - API密钥是否启用
- **有效时间** - API密钥的过期时间
- **备注** - API密钥的说明
- **操作** - 修改、删除等操作按钮

![API KEY列表](../../../../../images/user-guide/current-project/project-setting/project-setting-api.png)

### 修改API

1. 在API列表中找到要修改的API
2. 点击【修改】按钮
3. 修改API配置信息
4. 点击【确定】保存修改

### 删除API

1. 在API列表中找到要删除的API
2. 点击【删除】按钮
3. 确认删除操作

## 使用场景

API KEY用于以下场景：

- 第三方系统集成
- 自动化测试工具调用
- CI/CD流程中的API调用
- 自定义脚本访问项目数据

::: tip 安全提示
- API密钥具有配置的权限范围，请按需分配最小权限
- 不要将API密钥提交到代码仓库或公开分享
- 建议为不同的集成场景创建独立的API密钥
- 定期检查和清理不再使用的API密钥
- 可以设置有效时间来限制API密钥的使用期限
- 如果密钥泄露，请立即删除该API密钥
- 只有项目创建人和项目管理员可以管理API密钥
:::
