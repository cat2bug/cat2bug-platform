# 报告管理 [/project/report](/project/report)

报告是对团队、项目、测试用例、缺陷、交付物等工作管理做出的总结性报表，目前可通过手动或 Open API 方式创建。

## 什么是报告

测试报告是测试工作的重要产出物，用于总结测试执行情况、分析质量状况、提供决策依据。

**报告的作用：**

- 总结测试执行情况
- 分析缺陷分布和趋势
- 评估产品质量状况
- 为发布决策提供依据
- 记录测试过程和结果
- 提出改进建议

## 报告列表

报告列表展示了项目中已生成的所有测试报告，可以查看和管理报告。

![报告列表](../../../images/user-guide/current-project/report/report_list.png)

### 列表信息

每个报告显示以下信息：

- **报告类型** - 文本报告或实时报告
- **报告名称** - 报告的标题
- **报告时间** - 报告生成时间
- **数据来源** - 报告的数据来源
- **推送人** - 报告创建者
- **操作按钮** - 查看、导出、删除等操作

**报告类型说明：**
- **文本报告** - 手动创建或编辑的报告，内容为静态文本
- **实时报告** - 通过 API 自动生成的报告，可以实时更新数据

### 筛选功能

通过筛选条件快速找到需要的报告。

**按时间筛选：**
- 今天 - 今天创建的报告
- 本周 - 本周创建的报告
- 本月 - 本月创建的报告
- 自定义时间 - 指定时间范围

**按类型筛选：**
- 功能测试报告
- 性能测试报告
- 安全测试报告
- 验收测试报告
- 其他报告

**按创建人筛选：**
选择创建人，查看指定成员创建的报告。

### 搜索功能

在搜索框中输入关键字，快速搜索报告。

**搜索范围：**
- 报告名称
- 报告内容

**搜索特点：**
- 支持模糊匹配
- 实时搜索
- 高亮显示匹配内容

### 排序功能

点击列表表头，可以按不同字段排序。

**支持的排序字段：**
- 创建时间 - 按报告创建时间排序
- 更新时间 - 按报告更新时间排序
- 报告名称 - 按报告名称字母顺序排序

**排序方式：**
- 升序 - 从小到大、从旧到新
- 降序 - 从大到小、从新到旧

## 报告特点

### Markdown 格式

报告使用 Markdown 格式编写和展示，支持丰富的内容格式：

- 标题层级
- 列表和表格
- 图片和链接
- 代码块
- 引用和强调

### 数据可视化

报告支持多种图表展示：

- 饼图 - 展示分布占比
- 柱状图 - 展示数量对比
- 折线图 - 展示趋势变化
- 热力图 - 展示风险分布

### 多格式导出

报告可以导出为多种格式：

- **Word 格式** - 可编辑，适合内部流转
- **PDF 格式** - 格式固定，适合正式发布

## 最佳实践

### 报告编写建议

1. **客观准确**
   - 基于真实数据
   - 避免主观臆断
   - 数据来源可追溯

2. **重点突出**
   - 突出关键问题
   - 标注高风险项
   - 明确改进方向

3. **结论明确**
   - 明确测试是否通过
   - 说明是否可以发布
   - 给出具体建议

4. **图文并茂**
   - 使用图表展示数据
   - 添加截图说明问题
   - 保持格式统一

### 报告审核建议

1. **数据核对**
   - 核对统计数据是否准确
   - 检查计算公式是否正确
   - 确认数据来源可靠

2. **内容完整**
   - 检查是否包含所有必要内容
   - 确认结论是否明确
   - 验证建议是否可行

3. **格式规范**
   - 检查格式是否统一
   - 确认图表是否清晰
   - 验证链接是否有效

### 报告归档建议

1. **及时归档**
   - 测试完成后及时生成报告
   - 报告审核通过后立即归档
   - 定期整理历史报告

2. **分类存储**
   - 按项目分类
   - 按版本分类
   - 按时间分类

3. **权限管理**
   - 设置查看权限
   - 控制导出权限
   - 保护敏感信息

## 常见问题

### Q: 如何生成测试报告？

A: 目前主要通过 Open API 方式生成报告。可以编写脚本自动收集测试数据，调用 API 创建报告。未来版本将支持在界面上手动创建。

### Q: 报告可以修改吗？

A: 已生成的报告可以修改。但建议报告审核通过后不要随意修改，以保持数据的真实性和可追溯性。

### Q: 报告导出失败怎么办？

A: 可能的原因：
- 报告内容过大，导出超时
- 报告格式有误，无法转换
- 网络问题，导出中断

解决方法：
- 简化报告内容
- 检查 Markdown 格式
- 重试导出操作

### Q: 如何自动化生成报告？

A: 可以通过以下方式实现自动化：
1. 使用 Open API 获取测试数据
2. 编写脚本生成 Markdown 格式的报告内容
3. 调用 API 创建报告
4. 可以集成到 CI/CD 流程中

### Q: 报告可以分享给外部人员吗？

A: 可以。可以将报告导出为 PDF 格式，通过邮件或其他方式分享。也可以给外部人员分配"外部人员"角色，让他们在系统中查看报告。
