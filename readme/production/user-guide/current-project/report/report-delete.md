# 删除报告

删除不再需要的测试报告。

## 使用场景

- 删除测试报告草稿
- 清理过期的报告
- 删除错误的报告
- 测试数据清理

## 操作步骤

### 单个删除

#### 1. 找到要删除的报告

在报告列表中找到要删除的报告。

#### 2. 点击删除按钮

点击报告右侧的【删除】按钮。

![删除报告](../../../images/report_delete.png)

#### 3. 确认删除

在弹出的确认对话框中，确认删除操作。

#### 4. 完成删除

系统删除报告并刷新列表。

### 批量删除

#### 1. 选择报告

在报告列表中勾选要删除的多个报告。

#### 2. 点击批量删除

点击列表上方的【批量删除】按钮。

#### 3. 确认删除

在弹出的确认对话框中，确认删除操作。

**确认信息：**
- 显示选中的报告数量
- 列出报告名称
- 提示删除不可恢复

#### 4. 完成删除

系统批量删除报告并刷新列表。

## 删除影响

删除报告会导致：

- ❌ 报告内容永久删除，无法恢复
- ❌ 报告的评论一并删除
- ❌ 报告的导出记录删除
- ❌ 报告的分享链接失效

**不会影响：**
- ✅ 测试用例数据
- ✅ 缺陷数据
- ✅ 交付物数据
- ✅ 其他报告

## 删除前的考虑

### 是否需要归档

在删除前考虑是否需要归档保存：

**建议归档的报告：**
- 正式发布的测试报告
- 重要版本的测试报告
- 验收测试报告
- 包含重要决策的报告

**归档方式：**
1. 导出报告为 PDF 格式
2. 保存到本地或云盘
3. 记录报告的关键信息
4. 然后再删除系统中的报告

### 是否有人需要

确认是否有其他成员需要该报告：

**检查方式：**
- 查看报告的查看记录
- 查看报告的评论
- 询问团队成员
- 检查报告的分享记录

### 是否可以恢复

确认删除后是否可以重新生成：

**可以重新生成：**
- 通过 API 自动生成的报告
- 有生成脚本的报告
- 数据仍然存在的报告

**不能重新生成：**
- 手动编写的报告
- 数据已删除的报告
- 一次性的报告

## 权限说明

只有以下角色有删除权限：

- **项目创建人** - 可以删除所有报告
- **项目管理员** - 可以删除所有报告
- **报告创建人** - 可以删除自己创建的报告

**权限限制：**
- 普通成员不能删除他人创建的报告
- 外部人员不能删除报告
- 删除操作会记录操作日志

## 删除记录

系统会记录报告的删除操作：

**记录内容：**
- 删除时间
- 删除人
- 被删除的报告名称
- 删除原因（如填写）

**查看删除记录：**
1. 进入项目设置
2. 选择【操作日志】
3. 筛选删除操作
4. 查看删除记录

## 误删恢复

报告删除后无法直接恢复，但可以尝试以下方式：

### 从备份恢复

如果系统有定期备份：

1. 联系系统管理员
2. 说明需要恢复的报告
3. 提供报告的创建时间
4. 从备份中恢复数据

### 重新生成

如果报告是自动生成的：

1. 找到生成报告的脚本
2. 使用相同的参数
3. 重新调用 API 生成
4. 生成新的报告

### 从导出文件恢复

如果之前导出过报告：

1. 找到导出的文件
2. 复制报告内容
3. 通过 API 重新创建报告
4. 上传到系统

## 批量删除规则

批量删除时的处理规则：

**删除条件：**
- 只删除有权限删除的报告
- 跳过没有权限的报告
- 显示删除结果统计

**删除结果：**
- 成功删除的数量
- 失败的数量和原因
- 跳过的数量和原因

## 最佳实践

### 删除前确认

1. **确认报告不再需要**
   - 检查报告的使用情况
   - 询问团队成员意见
   - 确认没有依赖关系

2. **导出备份**
   - 重要报告先导出
   - 保存到安全位置
   - 记录报告信息

3. **填写删除原因**
   - 说明删除原因
   - 便于后续追溯
   - 避免误删

### 定期清理

1. **清理测试报告**
   - 删除草稿报告
   - 删除重复报告
   - 删除过期报告

2. **归档重要报告**
   - 导出正式报告
   - 保存到文档系统
   - 删除系统中的副本

3. **保持列表清爽**
   - 定期整理报告列表
   - 删除不需要的报告
   - 提高查找效率

::: tip 提示
1. 删除操作不可恢复，请谨慎操作
2. 重要报告建议先导出备份再删除
3. 批量删除时会跳过没有权限的报告
4. 删除操作会记录在操作日志中
5. 建议定期清理过期报告，保持列表整洁
:::
