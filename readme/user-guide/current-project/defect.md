# 缺陷管理 [/project/defect](/project/defect)

缺陷是测试工作中记录问题的基础单元，每个产品需求、工作任务、软件 BUG，都统一叫做缺陷，每个缺陷的生命周期如下图所示：

![测试流程](./images/defect-flow.png)

## 术语表

| 术语 | 解释 |
|------|------|
| 缺陷 | 需求、任务、BUG 统称缺陷 |
| 存活时间 | 缺陷从最近一次开始时间算起，直到结束或至今的时间区间 |
| 驳回次数 | 缺陷从最近一次开始时间算起，直到结束或至今的时间区间内，反复被驳回的次数总和 |

## 缺陷列表

缺陷列表是展示缺陷数据的表格，可以通过查询、排序、定义显示列等方式查询显示数据。

![缺陷列表](./images/defect-list.png)

::: tip 提示
在列表中每个缺陷的标题列中，统计了存活时间、驳回次数等指标；并在标题上方实时显示查看操作此缺陷的成员头像。
:::

## 页标签

页标签用于查看筛选需要显示的缺陷列表，默认有【我的】和【全部缺陷】两个页标签，用户也可以根据自己需求自定义添加，如添加：我关注的、我负责的、我待处理的等。点击页标签最右侧的【+】按钮，进行标签添加，如下图：

![新建缺陷标签](./images/defect-tabs.png)

## 创建缺陷

在缺陷列表页面，点击【新建缺陷】按钮，从右侧弹出新建对话框，填写相关数据后点击【新建】按钮保存提交，如下图：

![新建缺陷](./images/defect-create.png)

::: tip 提示
1. 在新建缺陷中，用例与交付物关联，如没有选择交付物，将无法显示可选择的用例；
2. 用例选择后，可【点击】选择任意步骤，代表在此步骤未通过测试。
:::

**创建步骤：**

1. **填写基本信息**
   - 缺陷标题（必填）
   - 缺陷描述（必填）
   - 缺陷类型（功能、UI、性能、安全等）
   - 优先级（紧急、高、中、低）

2. **关联信息**
   - 选择交付物（必填）
   - 关联测试用例（可选）
   - 选择用例步骤（可选）

3. **指派处理人**
   - 选择负责修复的开发人员

4. **上传附件**
   - 上传截图、日志等相关文件

5. 点击【新建】按钮保存提交

## 修改缺陷

修改用于调整缺陷的信息，一般由缺陷创建人、测试、项目管理员来维护，操作界面如下图：

![修改缺陷](./images/defect-modify.png)

**可修改内容：**
- 缺陷标题和描述
- 缺陷类型和优先级
- 关联的交付物和测试用例
- 处理人
- 附件

## 开启缺陷

当一个缺陷处理完被关闭后，在后续测试过程中，又再次发现，可改变其状态，使之变成开启状态，操作界面如下图：

![开启缺陷](./images/defect-start.png)

**操作步骤：**
1. 在缺陷列表中找到已关闭的缺陷
2. 点击【开启】按钮
3. 填写开启原因
4. 确认开启

## 指派缺陷

当想改变缺陷处理人时，可通过【指派】功能改变当前处理人，操作界面如下图：

![指派缺陷](./images/defect-assign.png)

**操作步骤：**
1. 选择要指派的缺陷
2. 点击【指派】按钮
3. 选择新的处理人
4. 填写指派说明（可选）
5. 确认指派

## 修复缺陷

当缺陷被提出后，根据缺陷提示修复完缺陷后，点击【修复】按钮，将修复后的缺陷转给测试人员，操作界面如下图：

![修复缺陷](./images/defect-repair.png)

**操作步骤：**
1. 开发人员修复代码
2. 点击【修复】按钮
3. 填写修复说明
4. 选择测试人员
5. 提交修复

## 驳回缺陷

当开发人员修复缺陷后，测试人员进行回归测试，如发现问题依旧没有解决，将会驳回此缺陷，操作界面如下图：

![驳回缺陷](./images/defect-reject.png)

**操作步骤：**
1. 测试人员验证修复结果
2. 发现问题未解决
3. 点击【驳回】按钮
4. 填写驳回原因（必填）
5. 上传验证截图（可选）
6. 确认驳回

## 通过缺陷

当开发人员修复缺陷后，测试人员进行回归测试，如发现问题已经解决，将会通过此缺陷，操作界面如下图：

![通过缺陷](./images/defect-pass.png)

**操作步骤：**
1. 测试人员验证修复结果
2. 确认问题已解决
3. 点击【通过】按钮
4. 填写验证说明（可选）
5. 确认通过

## 关闭缺陷

当缺陷问题已经解决，或者当前不再考虑处理此问题，可直接将其关闭，操作界面如下图：

![关闭缺陷](./images/defect-close.png)

**操作步骤：**
1. 选择要关闭的缺陷
2. 点击【关闭】按钮
3. 选择关闭原因（已解决、不予修复、重复缺陷等）
4. 填写关闭说明
5. 确认关闭

## 删除缺陷

删除后的缺陷信息将无法找回，一般只有创建错误或确认无效的缺陷，才可以删除。

**注意事项：**
- 删除操作不可恢复
- 只有缺陷创建人、测试人员、项目管理员有删除权限
- 建议使用"关闭"而不是"删除"

## 数据统计

系统提供了多种缺陷统计模版，点击缺陷列表右上角的【+】图标，跳转到缺陷模版页面，可选择显示的统计模版。

::: tip 提示
可通过拖动操作调整模版显示顺序
:::

![统计模版](./images/defect-statistics-option.png)

当设置完模版后，返回缺陷列表页面，即可显示所有设定的缺陷统计数据，如下图：

![带统计的缺陷列表](./images/defect-list-statistics.png)

::: tip 提示
多数统计面板中的参数，都可以与列表筛选联动，点击某个数据后，可按照点击数据进行筛选
:::

**统计模版类型：**

1. **缺陷趋势图** - 显示缺陷数量随时间的变化
2. **状态分布** - 显示各状态缺陷的数量
3. **优先级分布** - 显示各优先级缺陷的数量
4. **类型分布** - 显示各类型缺陷的数量
5. **处理人统计** - 显示各成员处理的缺陷数量
6. **模块分布** - 显示各交付物的缺陷数量

## 最佳实践

### 缺陷描述规范

**好的缺陷描述应包含：**
1. 问题现象 - 发生了什么
2. 重现步骤 - 如何重现问题
3. 预期结果 - 应该是什么样
4. 实际结果 - 实际是什么样
5. 环境信息 - 操作系统、浏览器版本等

**示例：**
```
标题：登录页面用户名输入框无法输入中文

描述：
1. 问题现象：在登录页面的用户名输入框中无法输入中文字符
2. 重现步骤：
   - 打开登录页面
   - 点击用户名输入框
   - 切换到中文输入法
   - 尝试输入中文
3. 预期结果：应该能够输入中文字符
4. 实际结果：无法输入任何中文字符
5. 环境：Windows 10, Chrome 120.0
```

### 优先级设置建议

- **紧急** - 系统崩溃、数据丢失、安全漏洞
- **高** - 核心功能无法使用、严重影响用户体验
- **中** - 一般功能问题、可以绕过的问题
- **低** - UI 细节问题、优化建议

### 缺陷跟踪建议

- 及时更新缺陷状态
- 添加详细的修复说明和验证说明
- 使用评论功能进行沟通
- 关联相关的测试用例和文档

## 常见问题

### Q: 如何批量操作缺陷？

A: 在缺陷列表中勾选多个缺陷，然后点击批量操作按钮（如批量指派、批量关闭等）。

### Q: 缺陷可以关联多个测试用例吗？

A: 目前一个缺陷只能关联一个测试用例。如果需要关联多个，可以在缺陷描述中添加其他用例的链接。

### Q: 如何查看我负责的缺陷？

A: 可以使用页标签功能，创建一个"我负责的"标签，设置筛选条件为"处理人=我"。

### Q: 缺陷可以导出吗？

A: 可以。在缺陷列表页面点击【导出】按钮，可以导出为 Excel 格式。

### Q: 如何查看缺陷的历史记录？

A: 打开缺陷详情页，可以看到缺陷的所有操作历史，包括创建、修改、状态变更等。
