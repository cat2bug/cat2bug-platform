# 文档管理 [/project/document](/project/document)

文档模块用于备份项目相关文件，目前可通过创建、更新、删除进行管理。

![文档列表](./images/doc-list.png)

## 文档概述

在项目开发和测试过程中，会产生大量的文档资料，如需求文档、设计文档、测试方案、会议纪要等。文档管理模块提供了一个集中存储和管理这些文档的地方。

**文档管理的作用：**

- 集中存储项目文档
- 方便团队成员查阅
- 保留文档历史版本
- 支持文档分类管理
- 便于文档检索

## 文档列表

文档列表展示了项目中的所有文档，支持多种查看和操作方式。

**列表功能：**

- **列表视图** - 以列表形式展示文档
- **卡片视图** - 以卡片形式展示文档
- **筛选功能** - 按文档类型、创建人、时间筛选
- **搜索功能** - 快速搜索文档标题和内容
- **排序功能** - 按名称、时间、大小排序
- **批量操作** - 批量下载、删除文档

## 创建文档

### 上传文档

支持上传多种格式的文档文件。

**上传步骤：**

1. 点击【上传文档】按钮
2. 选择要上传的文件
3. 填写文档信息
   - 文档标题（必填）
   - 文档描述
   - 文档类型（需求文档、设计文档、测试文档等）
   - 文档标签
4. 点击【确定】完成上传

**支持的文件格式：**

- **文档类** - Word (.doc, .docx), PDF (.pdf), Excel (.xls, .xlsx), PPT (.ppt, .pptx)
- **文本类** - TXT (.txt), Markdown (.md)
- **图片类** - JPG, PNG, GIF, BMP
- **压缩包** - ZIP, RAR, 7Z
- **其他** - 根据系统配置支持更多格式

**文件大小限制：** 默认单个文件最大 500MB

### 在线创建文档

支持在线创建 Markdown 格式的文档。

**创建步骤：**

1. 点击【新建文档】按钮
2. 选择【Markdown 文档】
3. 填写文档标题
4. 使用 Markdown 编辑器编写内容
5. 点击【保存】完成创建

**Markdown 编辑器功能：**

- 实时预览
- 语法高亮
- 快捷工具栏
- 支持插入图片、表格、代码块
- 支持导出为 HTML、PDF

## 查看文档

### 在线预览

支持在线预览多种格式的文档，无需下载。

**支持预览的格式：**

- PDF 文档
- 图片文件
- Markdown 文档
- 文本文件
- Office 文档（需要配置在线预览服务）

### 下载文档

点击文档列表中的【下载】按钮，可以下载文档到本地。

**批量下载：**

1. 勾选要下载的多个文档
2. 点击【批量下载】按钮
3. 系统会将文档打包成 ZIP 文件
4. 下载 ZIP 文件到本地

## 更新文档

### 更新文档内容

可以上传新版本的文档，系统会保留历史版本。

**更新步骤：**

1. 在文档列表中找到要更新的文档
2. 点击【更新】按钮
3. 上传新版本的文件
4. 填写更新说明
5. 点击【确定】完成更新

### 修改文档信息

可以修改文档的标题、描述、类型、标签等信息。

**修改步骤：**

1. 在文档列表中找到要修改的文档
2. 点击【编辑】按钮
3. 修改文档信息
4. 点击【保存】

## 删除文档

删除不再需要的文档。

**删除步骤：**

1. 在文档列表中找到要删除的文档
2. 点击【删除】按钮
3. 确认删除操作

**注意事项：**

- 删除操作不可恢复
- 删除文档会同时删除所有历史版本
- 只有文档创建人、项目管理员、团队管理员有删除权限

## 文档分类

### 文档类型

系统预定义了常用的文档类型：

- **需求文档** - 产品需求、功能需求
- **设计文档** - 架构设计、详细设计
- **测试文档** - 测试方案、测试报告
- **会议纪要** - 会议记录、决策记录
- **技术文档** - API 文档、部署文档
- **其他文档** - 其他类型的文档

### 文档标签

可以为文档添加标签，便于分类和检索。

**标签使用建议：**

- 使用简短的关键词
- 一个文档可以有多个标签
- 使用统一的标签命名规范
- 定期整理和合并重复标签

**示例标签：**
- 版本标签：v1.0, v2.0
- 状态标签：草稿, 待审核, 已发布
- 优先级标签：重要, 紧急
- 模块标签：用户模块, 订单模块

## 文档版本管理

### 版本历史

系统会自动保存文档的历史版本，可以查看和恢复历史版本。

**查看版本历史：**

1. 打开文档详情页
2. 点击【版本历史】标签
3. 查看所有历史版本
4. 每个版本显示：
   - 版本号
   - 更新时间
   - 更新人
   - 更新说明

### 版本对比

可以对比不同版本的文档内容，查看变更内容。

**对比步骤：**

1. 在版本历史中选择两个版本
2. 点击【对比】按钮
3. 查看差异内容
4. 支持高亮显示变更部分

### 版本恢复

可以恢复到历史版本。

**恢复步骤：**

1. 在版本历史中找到要恢复的版本
2. 点击【恢复】按钮
3. 确认恢复操作
4. 系统会创建一个新版本，内容为历史版本的内容

## 最佳实践

### 文档命名建议

1. **使用清晰的名称**
   - 包含文档类型
   - 包含版本号
   - 包含日期

2. **命名示例**
   ```
   好的命名：
   - 用户管理模块需求文档_v1.0_20240101
   - 系统架构设计_v2.0_20240115
   - 测试报告_第一轮_20240120

   不好的命名：
   - 文档1
   - 需求
   - test
   ```

### 文档组织建议

1. **按类型分类**
   - 使用文档类型字段
   - 使用标签辅助分类
   - 定期整理归档

2. **版本管理**
   - 重要变更时更新版本
   - 填写详细的更新说明
   - 保留关键版本

3. **权限控制**
   - 敏感文档限制访问
   - 定期 review 权限设置
   - 及时回收离职人员权限

### 文档维护建议

1. **定期清理**
   - 删除过期文档
   - 归档历史文档
   - 合并重复文档

2. **保持更新**
   - 及时更新文档内容
   - 同步最新的需求变更
   - 补充遗漏的文档

3. **质量控制**
   - 文档格式统一
   - 内容准确完整
   - 定期 review 文档质量

## 常见问题

### Q: 支持哪些文件格式？

A: 支持常见的文档格式（Word、PDF、Excel、PPT）、文本格式（TXT、Markdown）、图片格式（JPG、PNG）、压缩包（ZIP、RAR）等。具体支持的格式可以在系统设置中查看。

### Q: 文件大小有限制吗？

A: 有限制。默认单个文件最大 500MB。如果需要上传更大的文件，请联系系统管理员调整配置。

### Q: 如何批量上传文档？

A: 可以将多个文件打包成 ZIP 文件，然后上传 ZIP 文件。系统会自动解压并创建多个文档。

### Q: 文档可以分享给外部人员吗？

A: 目前外部人员角色不能查看文档。如果需要分享，可以下载文档后通过其他方式（邮件、网盘等）分享。

### Q: 如何恢复误删的文档？

A: 删除操作不可恢复。建议在删除前确认文档确实不再需要。重要文档建议定期备份到本地或其他存储系统。

### Q: 文档可以关联到缺陷或测试用例吗？

A: 目前不支持直接关联。但可以在缺陷或测试用例的描述中添加文档链接，或者使用相同的标签来建立关联关系。
