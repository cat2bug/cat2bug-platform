# 通知 [/notice/index](/notice/index)

## 功能概述

通知功能帮助用户及时接收项目相关的消息和系统通知。用户可以查看通知列表，并根据个人需求自定义通知设置。

## 通知列表

### 访问通知

1. 点击顶部导航栏的通知图标
2. 或访问 `/notice/index` 页面

### 通知类型

通知列表会显示以下类型的消息：

- **缺陷消息**：指向自己的缺陷相关通知
  - 缺陷被分配给你
  - 缺陷状态变更
  - 缺陷评论和更新
  
- **系统消息**：系统级别的通知
  - 团队邀请
  - 项目变更
  - 系统公告

### 通知操作

- 点击通知可查看详情
- 标记为已读
- 删除通知
- 批量操作

## 通知设置

通知设置允许用户自定义接收通知的类型和方式，确保只接收关心的信息。

### 发送选项

发送选项用于设置接收的通知类型和触发方式。

#### 缺陷通知

**通知事件：**
- ☑ 指派给我的：当缺陷被分配给你时接收通知
- ☑ 我关注的：当关注的缺陷有更新时接收通知

**触发方式：**
- ☑ 实时触发：事件发生时立即发送通知

#### 报告通知

**通知事件：**
- 有新报告时通知我：当项目生成新报告时接收通知

**触发方式：**
- 实时触发：报告生成后立即发送通知

### 接收平台

接收平台用于设置接收消息的渠道，支持多种通知方式。

#### 可选平台

- **系统内部通知**：在 Cat2Bug 系统内接收通知
  - 显示在通知列表中
  - 实时推送提醒
  
- **电子邮件**：通过邮件接收通知
  - 需要在个人中心配置邮箱地址
  - 支持邮件摘要
  
- **钉钉**：通过钉钉接收通知
  - 需要绑定钉钉账号
  - 支持钉钉机器人推送
  
- **飞书**：通过飞书接收通知
  - 需要绑定飞书账号
  - 支持飞书机器人推送
  
- **企业微信**：通过企业微信接收通知
  - 需要绑定企业微信账号
  - 支持企业微信应用推送

## 配置通知设置

### 打开通知设置

1. 访问通知页面 `/notice/index`
2. 点击"通知设置"按钮
3. 进入通知设置对话框

### 配置发送选项

1. 在"发送选项"标签页中
2. 选择要接收的通知类型：
   - 勾选"指派给我的"接收分配的缺陷通知
   - 勾选"我关注的"接收关注缺陷的更新
3. 选择触发方式：
   - 勾选"实时触发"立即接收通知
4. 配置报告通知选项

### 配置接收平台

1. 切换到"接收平台"标签页
2. 选择要启用的通知渠道：
   - 勾选"系统内部通知"在系统内接收
   - 勾选"电子邮件"通过邮件接收
   - 勾选"钉钉"通过钉钉接收
   - 勾选"飞书"通过飞书接收
   - 勾选"企业微信"通过企业微信接收
3. 点击"保存"按钮

> **提示**：使用第三方平台（钉钉、飞书、企业微信）接收通知前，需要先在个人中心或项目设置中完成相应平台的账号绑定和配置。

## 最佳实践

## 常见问题

**Q: 为什么收不到通知？**  
A: 请检查通知设置是否正确配置，确认已勾选相应的通知类型和接收平台。

**Q: 如何关闭某类通知？**  
A: 在通知设置的发送选项中，取消勾选不想接收的通知类型。

**Q: 可以同时使用多个接收平台吗？**  
A: 可以，您可以同时启用多个接收平台，系统会向所有选中的平台发送通知。

**Q: 第三方平台通知需要什么配置？**  
A: 需要在个人中心绑定对应平台的账号，或在项目设置中配置机器人。

**Q: 通知会保留多久？**  
A: 系统通知会长期保留，您可以随时查看历史通知。

**Q: 如何只接收重要通知？**  
A: 在发送选项中只勾选"指派给我的"，取消其他选项，可以减少通知数量。

**Q: 邮件通知的频率如何？**  
A: 如果选择"实时触发"，每次事件发生都会发送邮件；您也可以选择其他触发方式来控制频率。

**Q: 可以为不同项目设置不同的通知方式吗？**  
A: 当前通知设置是全局的，适用于所有项目。
