# 电子邮件通知

## 概述

电子邮件通知允许用户通过邮件接收系统通知，适合在非工作时间或离开系统时接收重要消息。

## 功能特点

- **邮件推送**：通知会发送到用户的邮箱
- **邮件摘要**：支持邮件摘要功能
- **离线接收**：即使不在系统中也能收到通知

## 前置条件

### 用户配置

- 需要在个人中心配置邮箱地址
- 确保邮箱地址有效且可以正常接收邮件

### 系统配置

- 需要系统管理员在后台设置邮件服务器配置
- 邮件服务器必须正确配置才能发送邮件

## 启用邮件通知

1. 访问通知页面 `/notice/index`
2. 点击"通知设置"按钮
3. 切换到"接收平台"标签页
4. 勾选"电子邮件"
5. 点击"保存"按钮

## 邮件内容

邮件通知包含以下信息：

- 通知标题
- 通知内容摘要
- 相关链接（可直接跳转到系统）
- 发送时间

## 最佳实践

- 确保邮箱地址正确且可用
- 非工作时间启用邮件通知
- 定期检查邮箱，避免遗漏重要通知
- 将系统邮件地址加入白名单，避免被误判为垃圾邮件

## 常见问题

**Q: 为什么收不到邮件通知？**  
A: 请检查以下几点：
- 个人中心是否配置了正确的邮箱地址
- 系统管理员是否已配置邮件服务器
- 检查邮箱的垃圾邮件文件夹
- 确认通知设置中已勾选"电子邮件"

**Q: 邮件通知的频率如何？**  
A: 如果选择"实时触发"，每次事件发生都会发送邮件；您也可以选择其他触发方式来控制频率。

**Q: 可以修改邮件通知的内容吗？**  
A: 邮件通知的内容由系统自动生成，用户无法自定义。

**Q: 邮件通知会延迟吗？**  
A: 邮件发送可能会有轻微延迟，具体取决于邮件服务器的性能和网络状况。

**Q: 如何将系统邮件加入白名单？**  
A: 在您的邮箱设置中，将系统发件地址添加到联系人或白名单中，具体操作请参考您使用的邮箱服务商的帮助文档。
